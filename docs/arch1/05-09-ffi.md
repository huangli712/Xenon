## 1. 模块概述

### 1.1 核心角色

FFI（Foreign Function Interface）集成模块是 Xenon 与上游 C 库（特别是 BLAS/LAPACK）互操作的桥梁。它提供零开销的原始指针访问接口，使得上游库能够直接操作张量的底层内存，而无需进行数据拷贝。

| 影响领域 | 说明 |
|----------|------|
| **BLAS 集成** | 上游线性代数库通过指针 API 直接访问张量数据 |
| **C 互操作** | 与 C 语言库的无缝对接 |
| **性能关键路径** | 零开销抽象，无运行时检查 |
| **安全性边界** | 明确定义 unsafe 代码的契约和前提条件 |

### 1.2 设计目标

1. **零开销**：FFI API 不引入额外的内存分配或数据拷贝
2. **BLAS 友好**：提供完整的 BLAS 兼容性检查和布局查询
3. **安全边界清晰**：所有 unsafe 函数有详尽的 Safety 文档
4. **类型安全**：利用 Rust 类型系统区分可变/不可变访问

### 1.3 在架构中的位置

```
依赖层级：

L4: ffi ←── tensor, layout
         ↓
L5: 上游库（BLAS bindings, LAPACK bindings, 自定义 C 库）
```

FFI 模块位于架构顶层，依赖张量核心模块和布局模块，为上游库提供接口。

### 1.4 范围界定

| 范围内 | 范围外 |
|--------|--------|
| 指针访问 API | BLAS/LAPACK 绑定实现 |
| BLAS 兼容性查询 | GPU 内存操作 |
| 原始部件构造/解构 | 序列化/反序列化 |
| 索引转换工具 | 跨进程共享内存 |

### 2.1 模块组织

FFI 模块采用单文件设计，原因如下：

1. **API 内聚性高**：所有 FFI 相关功能紧密相关

## 3. 指针 API 设计

### 3.1 as_ptr()

返回数据起始位置的不可变原始指针。

```rust
impl<S, D, A> TensorBase<S, D>
where
    S: Storage<Elem = A>,
    D: Dimension,
{
    /// 返回数据起始位置的不可变原始指针。
    ///
    /// 指针指向第一个逻辑元素（考虑 offset）。
    /// 返回的指针在 `self` 被修改或释放后无效。
    ///
    /// # Returns
    ///
    /// 指向第一个逻辑元素的 `*const A` 指针。
    ///
    /// # Example
    ///
    /// ```ignore
    /// let tensor = Tensor2::<f64>::zeros([3, 4]);
    /// let ptr = tensor.as_ptr();
    /// // 可安全传递给 C 函数
    /// ```
    pub fn as_ptr(&self) -> *const A {
        unsafe {
            self.storage.as_ptr().add(self.offset)
        }
    }
}
```

### 3.2 as_mut_ptr()

返回数据起始位置的可变原始指针。

```rust
impl<S, D, A> TensorBase<S, D>
where
    S: StorageMut<Elem = A>,
    D: Dimension,
{
    /// 返回数据起始位置的可变原始指针。
    ///
    /// 仅对可写存储（Owned、ViewMut、ArcRepr）可用。
    /// 返回的指针在 `self` 被修改或释放后无效。
    ///
    /// # Returns
    ///
    /// 指向第一个逻辑元素的 `*mut A` 指针。
    ///
    /// # Safety
    ///
    /// 调用方须保证：
    /// - 指针使用期间底层数据保持有效
    /// - 无其他引用同时访问同一数据（别名规则）
    /// - 对于 ArcRepr，若引用计数 > 1，应先调用 make_mut()
    ///
    /// # Example
    ///
    /// ```ignore
    /// let mut tensor = Tensor2::<f64>::zeros([3, 4]);
    /// let ptr = tensor.as_mut_ptr();
    /// // 可安全传递给需要可变指针的 C 函数
    /// ```
    pub fn as_mut_ptr(&mut self) -> *mut A {
        unsafe {
            self.storage.as_mut_ptr().add(self.offset)
        }
    }
}
```

### 3.3 as_ptr_unchecked()

不检查偏移量有效性的指针访问。

```rust
impl<S, D, A> TensorBase<S, D>
where
    S: Storage<Elem = A>,
    D: Dimension,
{
    /// 不检查偏移量有效性的指针访问。
    ///
    /// 比 `as_ptr()` 更快，但跳过边界检查。
    /// 仅在性能关键路径且已确保安全的场景使用。
    ///
    /// # Safety
    ///
    /// 调用方须保证：
    /// - `self.offset < self.storage.len()` 成立
    /// - `self.storage.as_ptr()` 返回有效指针
    /// - 指针使用期间 `self` 保持有效
    /// - 所有可访问元素已正确初始化
    ///
    /// # Example
    ///
    /// ```ignore
    /// let tensor = Tensor1::<f64>::from_vec(vec![1.0, 2.0, 3.0]);
    /// let ptr = unsafe { tensor.as_ptr_unchecked() };
    /// ```
    #[inline]
    pub unsafe fn as_ptr_unchecked(&self) -> *const A {
        self.storage.as_ptr().add(self.offset)
    }
}
```

### 3.4 指针 API 使用场景

| API | 适用场景 | 安全级别 |
|-----|----------|----------|
| `as_ptr()` | 传递给只读 C 函数 | Safe |
| `as_mut_ptr()` | 传递给会修改数据的 C 函数 | Safe（调用时） |
| `as_ptr_unchecked()` | 性能关键路径，已验证安全 | Unsafe |

### 5.3 lda()

```rust
impl<S, D> TensorBase<S, D>
where
    D: Dimension,
{
    /// 返回 leading dimension（F-order 下为第一轴步长）。
    ///
    /// 仅对 2D 数组有意义。BLAS 函数使用 LDA 来计算矩阵元素的内存地址。
    ///
    /// # Returns
    ///
    /// - `Some(isize)`：2D 数组的 leading dimension（元素单位）
    /// - `None`：非 2D 数组
    ///
    /// # BLAS LDA 语义
    ///
    /// | 布局 | LDA 含义 |
    /// |------|----------|
    /// | F-order | 第一维长度（可能含 padding） |
    /// | C-order | 第二维长度（可能含 padding） |
    ///
    /// 对于 F-order 矩阵 `A[M, N]`，`LDA = stride[1]`，表示每列的元素间距。
    ///
    /// # Example
    ///
    /// ```ignore
    /// let a = Tensor2::<f64>::zeros([3, 4]);  // 3 rows, 4 cols
    /// assert_eq!(a.lda(), Some(3));  // F-order, LDA = M
    /// ```
    pub fn lda(&self) -> Option<isize> {
        if self.ndim() != 2 {
            return None;
        }
        // F-order: LDA = stride[1] (distance between columns)
        // C-order: LDA = stride[0] (distance between rows)
        let strides = self.strides();
        if self.is_f_contiguous() {
            Some(strides[1])
        } else if self.is_c_contiguous() {
            Some(strides[0])
        } else {
            // 非连续布局，返回主步长
            Some(strides[1].abs())
        }
    }
}
```

### 5.4 is_blas_compatible()

```rust
impl<S, D> TensorBase<S, D>
where
    D: Dimension,
{
    /// 检查内存布局是否可直接传递给 BLAS。
    ///
    /// # BLAS 兼容性条件
    ///
    /// | 条件 | 说明 |
    /// |------|------|
    /// | 连续性 | F-contiguous 或 C-contiguous |
    /// | 正步长 | 所有步长 > 0（无反转维度） |
    /// | 无零步长 | 无广播维度 |
    ///
    /// # Returns
    ///
    /// `true` 表示可直接传递给 BLAS；`false` 表示需要先复制到连续布局。
    ///
    /// # Example
    ///
    /// ```ignore
    /// let a = Tensor2::<f64>::zeros([3, 4]);
    /// assert!(a.is_blas_compatible());
    ///
    /// let b = a.slice(s![.., 1..3]);  // 非连续切片
    /// assert!(!b.is_blas_compatible());
    /// ```
    pub fn is_blas_compatible(&self) -> bool {
        // BLAS 要求：连续（F 或 C）、正步长、无零步长
        self.is_contiguous() 
            && !self.has_zero_stride()
            && !self.has_neg_stride()
    }
}
```

**BLAS 兼容性检查完整逻辑**：

```
function is_blas_compatible(tensor):
    // 检查 1: 连续性
    if not (is_f_contiguous(tensor) or is_c_contiguous(tensor)):
        return false
    
    // 检查 2: 正步长（无反转）
    for stride in tensor.strides:
        if stride < 0:
            return false
    
    // 检查 3: 无零步长（无广播）
    for stride in tensor.strides:
        if stride == 0:
            return false
    
    return true
```

### 5.5 blas_layout()

```rust
impl<S, D> TensorBase<S, D>
where
    D: Dimension,
{
    /// 返回 BLAS 布局标识。
    ///
    /// # Returns
    ///
    /// - `Some(BlasLayout::ColumnMajor)`：F-contiguous
    /// - `Some(BlasLayout::RowMajor)`：C-contiguous（非 F-contiguous）
    /// - `None`：不兼容 BLAS
    ///
    /// # Example
    ///
    /// ```ignore
    /// let a = Tensor2::<f64>::zeros([3, 4]);  // 默认 F-order
    /// assert_eq!(a.blas_layout(), Some(BlasLayout::ColumnMajor));
    /// ```
    pub fn blas_layout(&self) -> Option<BlasLayout> {
        if !self.is_blas_compatible() {
            return None;
        }
        if self.is_f_contiguous() {
            Some(BlasLayout::ColumnMajor)
        } else {
            Some(BlasLayout::RowMajor)
        }
    }
}
```

### 5.6 blas_trans()

```rust
impl<S, D> TensorBase<S, D>
where
    D: Dimension,
{
    /// 返回 BLAS Trans 参数。
    ///
    /// 基于当前布局与目标布局（默认 F-order）的关系。
    /// 如果数组是 C-contiguous，可传递 `Trans` 给期望 F-order 的 BLAS 函数，
    /// 以避免数据复制。
    ///
    /// # Returns
    ///
    /// - `BlasTrans::NoTrans`：F-contiguous，无需转置标识
    /// - `BlasTrans::Trans`：C-contiguous，使用转置标识避免复制
    /// - `BlasTrans::NoTrans`：非连续但兼容，返回 NoTrans（调用方自行处理）
    ///
    /// # 策略
    ///
    /// | 实际布局 | 期望布局 | blas_trans() | 效果 |
    /// |----------|----------|--------------|------|
    /// | F-contiguous | F-order | NoTrans | 直接使用 |
    /// | C-contiguous | F-order | Trans | 逻辑转置，无复制 |
    /// | 非连续 | - | NoTrans | 需调用方处理 |
    ///
    /// # Example
    ///
    /// ```ignore
    /// // 传递 C-order 矩阵给期望 F-order 的 BLAS 函数
    /// let a = Tensor2::<f64>::zeros_c([3, 4]);  // C-order
    /// let trans = a.blas_trans();  // Trans
    /// // 调用 dgemm(..., trans, ...) 而非复制数据
    /// ```
    pub fn blas_trans(&self) -> BlasTrans {
        if self.is_f_contiguous() {
            BlasTrans::NoTrans
        } else if self.is_c_contiguous() {
            BlasTrans::Trans
        } else {
            BlasTrans::NoTrans
        }
    }
}
```

## 6. 索引转换

### 6.1 index_to_offset()

```rust
impl<S, D> TensorBase<S, D>
where
    D: Dimension,
{
    /// 将多维索引转换为相对于数据起始位置的元素偏移量。
    ///
    /// # Arguments
    ///
    /// * `index` - 多维索引切片，长度须等于维度数
    ///
    /// # Returns
    ///
    /// 元素偏移量（可能为负数，当存在负步长时）。
    ///
    /// # Panics
    ///
    /// - 索引长度与维度数不匹配
    /// - 索引越界（任一维度）
    ///
    /// # Example
    ///
    /// ```ignore
    /// let tensor = Tensor2::<f64>::zeros([3, 4]);
    /// assert_eq!(tensor.index_to_offset(&[1, 2]), 7);  // 1*1 + 2*3
    /// ```
    pub fn index_to_offset(&self, index: &[usize]) -> isize {
        assert!(index.len() == self.ndim(), "index dimension mismatch");
        
        let shape = self.shape();
        let strides = self.strides();
        
        let mut offset: isize = 0;
        for i in 0..self.ndim() {
            assert!(index[i] < shape[i], "index out of bounds");
            offset += (index[i] as isize) * strides[i];
        }
        offset
    }
}
```

### 6.2 index_to_ptr()

```rust
impl<S, D, A> TensorBase<S, D>
where
    S: Storage<Elem = A>,
    D: Dimension,
{
    /// 将多维索引转换为对应元素的原始指针。
    ///
    /// # Arguments
    ///
    /// * `index` - 多维索引切片，长度须等于维度数
    ///
    /// # Returns
    ///
    /// 指向指定元素的 `*const A` 指针。
    ///
    /// # Panics
    ///
    /// - 索引长度与维度数不匹配
    /// - 索引越界
    ///
    /// # Safety
    ///
    /// 返回的指针在 `self` 被修改或释放后无效。
    ///
    /// # Example
    ///
    /// ```ignore
    /// let tensor = Tensor1::<i32>::from_vec(vec![10, 20, 30, 40]);
    /// let ptr = tensor.index_to_ptr(&[2]);
    /// assert_eq!(unsafe { *ptr }, 30);
    /// ```
    pub fn index_to_ptr(&self, index: &[usize]) -> *const A {
        let offset = self.index_to_offset(index);
        unsafe {
            self.as_ptr().offset(offset)
        }
    }
}
```

### 6.3 索引转换算法

**偏移量计算公式**：

```
offset = sum(stride[i] * index[i]) for all i in [0, ndim)
```

**示例**：

```
shape = [2, 3, 4], strides = [1, 2, 6] (F-order)
index = [1, 2, 1]

offset = 1*1 + 2*2 + 1*6 = 1 + 4 + 6 = 11
```

## 7. 原始部件构造与解构

### 7.1 from_raw_parts()

```rust
impl<'a, A, D> TensorBase<ViewRepr<&'a A>, D>
where
    D: Dimension,
{
    /// 从原始部件构造不可变视图。
    ///
    /// # Arguments
    ///
    /// * `ptr` - 数据起始指针（不可变）
    /// * `shape` - 各轴长度
    /// * `strides` - 各轴步长（元素单位，有符号）
    /// * `offset` - 数据起始偏移量（元素单位）
    ///
    /// # Returns
    ///
    /// 新的 `TensorView<'a, A, D>` 实例。
    ///
    /// # Safety
    ///
    /// 调用方须保证以下所有条件，否则导致未定义行为：
    ///
    /// | 前提条件 | 说明 |
    /// |----------|------|
    /// | **指针有效性** | `ptr` 须非空、非悬垂，且对齐到 `align_of::<A>()` |
    /// | **内存范围** | `ptr` 起始的内存范围须足够大，能覆盖所有可访问元素（考虑 offset、shape、strides） |

### 7.3 into_raw_parts()

```rust
impl<S, D, A> TensorBase<S, D>
where
    S: StorageIntoRaw<Elem = A>,
    D: Dimension,
{
    /// 消费数组，返回原始部件。
    ///
    /// 将张量的所有权转移给调用方，返回组成张量的原始部件。
    /// 调用方负责确保内存的安全释放。
    ///
    /// # Returns
    ///
    /// 元组 `(ptr, shape, strides, offset)`：
    /// - `ptr`: 数据指针（可变）
    /// - `shape`: 维度类型
    /// - `strides`: 步长类型
    /// - `offset`: 起始偏移量
    ///
    /// # Safety
    ///
    /// - 返回的指针所有权转移给调用方
    /// - 对于 Owned 存储，调用方须最终释放内存
    /// - 对于 View 存储，返回的指针生命周期与原借用绑定
    /// - 调用方须遵守别名规则
    ///
    /// # Example
    ///
    /// ```ignore
    /// let tensor = Tensor2::<f64>::zeros([3, 4]);
    /// let (ptr, shape, strides, offset) = tensor.into_raw_parts();
    /// // 调用方现在拥有 ptr，负责释放
    /// ```
    pub fn into_raw_parts(self) -> (*mut A, D, D, usize) {
        let ptr = self.storage.into_raw();
        let shape = self.shape;
        let strides = self.strides;
        let offset = self.offset;
        
        // 防止 drop
        std::mem::forget(self);
        
        (ptr, shape, strides, offset)
    }
}
```

## 8. Safety 文档规范

### 8.1 标准 Safety 节格式

所有 unsafe 函数必须包含 `# Safety` 节，格式如下：

```rust
/// 函数简短描述。
///
/// 详细描述...
///
/// # Arguments
///
/// * `arg1` - 参数说明
/// * `arg2` - 参数说明
///
/// # Returns
///
/// 返回值说明。
///
/// # Safety
///
/// 调用方须保证以下所有条件：
///
/// 1. **前提条件 1**：详细说明
/// 2. **前提条件 2**：详细说明
/// 3. **前提条件 3**：详细说明
///
/// 违反任一条件将导致未定义行为。
///
/// # Example
///
/// ```ignore
/// // 安全使用示例
/// ```
///
/// # Panics
///
/// （如有）panic 条件说明
```

### 8.2 Safety 前提条件分类

| 类别 | 示例 | 检查方式 |
|------|------|----------|
| **指针有效性** | 非空、对齐、非悬垂 | 调用方保证 |
| **内存范围** | 足够大覆盖所有访问 | 调用方保证 |
| **生命周期** | 数据在引用期间有效 | Rust 借用检查（部分） |
| **别名规则** | 无数据竞争 | 调用方保证 |
| **布局一致性** | shape/strides 匹配 | 可运行时断言 |
| **边界安全** | 索引不越界 | 可运行时检查 |
| **元素初始化** | 所有元素已初始化 | 调用方保证 |
| **线程安全** | Send/Sync 满足 | 类型系统（部分） |

### 8.3 各 unsafe 函数 Safety 清单

| 函数 | Safety 前提条件 |
|------|-----------------|
| `as_ptr_unchecked()` | offset 有效、指针有效、元素已初始化 |
| `from_raw_parts()` | 指针有效、内存范围、生命周期、别名（共享读）、布局一致、边界安全、元素初始化、线程安全 |
| `from_raw_parts_mut()` | 指针有效、内存范围、生命周期、别名（独占）、布局一致、边界安全、元素初始化、线程安全 |

## 12. 设计决策记录

### D1: 为什么 FFI 模块使用单文件设计？

**决策**: 使用单文件 `src/ffi.rs` 而非多文件模块。

**理由**:
1. **API 内聚性**：所有 FFI 相关功能紧密相关
2. **代码量适中**：预计约 300-400 行，无需拆分
3. **简化依赖**：避免过度模块化带来的复杂性
4. **参考 ndarray**：ndarray 的 FFI 相关代码也集中在少数文件

### D2: 为什么 BLAS 兼容性检查不自动复制？

**决策**: `is_blas_compatible()` 仅返回检查结果，不自动复制。

**理由**:
1. **显式优于隐式**：复制是昂贵操作，应由调用方显式决定
2. **性能可预测**：避免隐藏的性能开销
3. **灵活性**：调用方可能选择不同的处理策略
4. **与 ndarray 一致**：ndarray 也采用类似设计

### D3: 为什么 strides_bytes() 返回 & [isize] 而非 Vec？

**决策**: 返回切片引用而非新的 Vec。

**理由**:
1. **零分配**：避免每次调用都分配内存
2. **实现选项**：可使用缓存或栈上数组
3. **一致性**：与 `strides()` 返回类型一致

### D4: 为什么 lda() 仅支持 2D 数组？

**决策**: `lda()` 对非 2D 数组返回 `None`。

**理由**:
1. **BLAS 语义**：LDA（Leading Dimension）仅对矩阵有意义
2. **类型安全**：避免误用
3. **明确错误**：返回 `None` 比返回错误值更清晰

### D5: 为什么 blas_trans() 对复数返回 NoTrans 而非 ConjTrans？

**决策**: 当前版本 `blas_trans()` 不区分实数和复数。

**理由**:
1. **简化实现**：复数共轭转置需要元素类型信息
2. **需求明确**：需求文档未要求复数特殊处理
3. **扩展性**：可通过泛型特化在未来版本添加

### D6: 为什么 from_raw_parts 要求元素已初始化？

**决策**: Safety 文档要求所有可访问元素已正确初始化。

**理由**:
1. **Rust 语义**：读取未初始化内存是 UB
2. **视图可读**：创建的视图可能被读取
3. **与 MaybeUninit 兼容**：文档提及 MaybeUninit 语义
4. **安全边界清晰**：调用方负责保证初始化状态

### D7: strides_bytes 与 strides 的关系

**决策**: `strides_bytes[i] = strides[i] * size_of::<A>()`

**理由**:
1. **直接映射**：字节步长是元素步长的简单缩放
2. **BLAS 兼容**：某些 C 库需要字节步长
3. **类型安全**：区分元素单位和字节单位
