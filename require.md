版本: v21
日期: 2026-04-06
许可: MIT
MSRV: Rust 1.85+

# 第一部分：项目定位

## 1. 项目概述

Xenon 是一个 Rust 多维数组（张量）库，定位为科学计算的数值基础设施，类似于 Python 生态中 NumPy ndarray 层的多维数组基础设施。

### 1.1 目标用户

| 用户类型 | 核心诉求 |
|----------|----------|
| 库开发者 | 稳定API、高性能互操作 |
| 系统开发者 | no_std支持、底层内存控制、确定性数值行为、最小依赖 |
| 间接用户 | 性能、正确性、与Python经验的直觉一致性 |

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 正确性优先 | 类型安全、内存安全、数值精度 |
| 性能敏感 | 高性能抽象 |
| 清晰抽象 | API 语义明确，无隐式行为 |
| 科学计算导向 | BLAS 兼容 |
| 最小依赖 | 仅允许可选的并行（rayon）和SIMD相关（pulp）外部依赖 |

### 1.3 工程约束

| 约束 | 要求 |
|------|------|
| Crate 结构 | 单 crate，遵循 SemVer |
| no_std 支持 | 支持 `no_std`，`std` 为默认 |

---

## 2. 项目范围

### 2.1 范围内

- N维数组的存储、构造
- N维数组的索引操作、切片操作、形状操作、归约操作、广播操作、集合操作
- N维数组的逐元素运算
- N维数组的显式类型转换
- 向量内积
- 原始指针 API，供上游库零开销集成
- 自定义复数类型
- 临时工作空间（供上游库使用）

### 2.2 范围外

- 矩阵-矩阵乘法、矩阵分解、对角化等高级线性代数
- 快速傅里叶变换、稀疏矩阵、自动微分、随机数
- BLAS/LAPACK 绑定（由上游库通过指针API集成）
- GPU后端
- serde序列化
- arena分配器
- 栈分配小数组

# 第二部分：功能需求

## 3. 维度系统

- 须支持0至6维静态维度及动态维度
- 须提供维度类型之间的转换
- 须提供从常见Rust类型构造维度类型的能力
- 须支持负步长

---

## 4. 元素类型

- 须支持以下元素类型：整数（i32、i64）、浮点（f32、f64）、复数（Complex<f32>、Complex<f64>）、bool、usize
- 须按运算能力对类型进行分层约束：基础操作（所有类型）、四则运算（整数、浮点、复数）、数学函数（仅实数）、复数运算（仅复数）
- 不支持自动类型提升，类型转换须显式
- 元素类型集合须为封闭集合，不支持下游扩展

---

## 5. 复数类型

- 须提供自定义复数类型，不依赖外部 crate
- 须支持复数算术运算
- 须支持复数与同精度实数的混合运算，跨精度须显式转换
- 须提供近似相等判断方法
- 须支持格式化输出
- 内存布局须与 C99 `_Complex` 兼容

---

## 6. 存储系统

- 须提供多级别的数据访问控制(只读、可写、拥有)
- 须支持数据的多种持有与访问方式(持有、只读引用、可写引用、共享引用)
- 涉及零大小类型和空数组的操作不得引发未定义行为

---

## 7. 内存布局

- 只支持列优先（F-order）布局
- 行优先（C-order）布局不在当前范围内
- 须支持对齐布局，逻辑数据须与未对齐时完全一致
- 须提供布局状态查询能力

---

## 8. 张量类型

- 须提供以存储模式和维度类型为参数的张量类型
- 须提供常用张量类型的便捷别名
- 须提供形状、维度、步长、布局、数据位置等基础信息的查询能力
- 所有构造路径须验证合法性，防止越界访问

---

# 第三部分：计算需求

## 9. 计算加速

### 9.1 SIMD

- 须提供可选的SIMD加速功能，默认关闭
- 须为逐元素运算、归约、内积等操作提供加速路径
- 不满足SIMD条件时须自动回退
- SIMD路径的计算结果须与标量路径一致

### 9.2 并行

- 须提供可选的数据并行功能，默认关闭
- 并行阈值可配置
- 须支持逐元素运算、归约、函数映射、多数组同步操作的并行执行
- 禁止嵌套并行

### 9.3 自动路径选择

- 须根据数据规模和可用硬件能力自动选择串行或并行执行路径

---

## 10. 线程安全

- 所有存储模式须满足线程安全要求，可在多线程间安全传递和共享

---

## 11. 迭代器

- 须支持按元素、按轴、按窗口、按索引遍历，以及多数组同步遍历
- 所有迭代器须提供精确的元素数量信息
- 空数组迭代须立即结束
- 填充数组的迭代须仅遍历逻辑元素
- 禁止对广播结果进行可变迭代
- 零维张量元素迭代产出恰好 1 个元素，不支持按轴和按窗口遍历

---

# 第四部分：API 需求

## 12. 逐元素运算

- 须支持算术运算（加、减、乘、除），适用于数值类型（整数、浮点、复数）
- 须支持一元运算：绝对值、取反、符号函数、平方
- 须支持数学函数（仅实数类型 f32/f64）：正弦、平方根、指数、自然对数、下取整、上取整
- 须支持复数运算（模、共轭），模运算结果须为实数类型
- 须支持逻辑非（仅 bool）
- 须支持比较运算（等于、不等于、小于、大于），返回 bool 类型张量，NaN 比较遵循 IEEE 754
- 须支持标量与张量的逐元素运算
- 所有二元运算和比较运算须支持广播

## 13. 矩阵运算

- 须支持向量内积
- 复数内积须满足共轭线性的数学定义
- 维度或形状不匹配时须提供可恢复的错误处理路径

## 14. 归约

- 须支持sum归约
- 须支持全局归约和沿指定轴归约
- 归约须支持保留被归约轴为长度 1 的选项
- 整数归约溢出时视为不可恢复错误
- 空数组 sum 归约返回加法单位元

## 15. 集合操作

- 须提供唯一值操作，返回排序后的不重复元素
- 唯一值操作须对浮点类型和复数类型可用
- 须支持按自定义键去重
- bool 和 usize 不适用

## 16. 广播

- 须遵循 NumPy 广播规则
- 广播操作不得拷贝数据
- 广播结果须为只读
- 须提供显式广播方法
- 须提供可恢复的错误处理路径

## 17. 形状操作

- 须提供转置操作，不得拷贝数据
- 须提供 reshape 操作

## 18. 索引操作

- 须支持多维整数索引、范围索引（切片）、布尔掩码高级索引
- 布尔掩码索引须返回一维数组，始终产生数据拷贝

## 19. 构造操作

- 须提供全零、全一、单位矩阵等标准构造方法
- 须支持从动态数组、切片、固定数组、标量值构造张量
- 嵌套集合构造须验证内层一致性

## 20. 运算符重载

- 须支持四则运算的运算符语法，并支持广播
- 须支持张量与张量、张量与标量之间的运算
- 所有组合均产生新的独立张量
- 四则运算以外的运算符语法不在当前范围内

## 21. 实用操作

- 须提供裁剪操作（将元素限制在 [min, max] 范围，NaN 保持不变，min > max 时视为不可恢复错误）
- 须提供填充操作（填充所有逻辑元素，广播结果时视为不可恢复错误）

## 22. 连续性保证

- 须提供确保数据在内存中连续存储的操作
- 须同时支持数据在内存中非完全连续但仍可正确访问和操作的场景

## 23. 类型转换

- 须提供逐元素类型转换方法，溢出行为须明确定义
- 须同时提供同类型拷贝方法
- 类型转换仅适用于持有数据的存储模式
- 须支持各存储模式之间的转换
- 张量类型须支持标准库的类型转换接口

## 24. 格式化输出

- 面向用户的格式化输出须简洁可读
- 数据展示须遵循 NumPy 风格
- 超过指定元素数量的数组须触发输出截断

## 25. FFI 集成

- 须提供原始指针 API，供上游库零开销集成
- 须支持从裸指针构造张量和将张量解构为裸指针
- 须提供 BLAS 兼容性相关 API
- 须提供多维索引到指针和偏移量的转换

## 26. 临时工作空间

- 须提供临时工作空间类型，支持对齐分配
- 须支持 no_std 环境
- 须支持工作空间的分割与动态扩容
- 扩容操作须保证不违反已有的数据引用安全性

# 第五部分：工程保障

## 27. 错误处理

- 可恢复错误须以返回值形式报告
- 不可恢复错误须终止程序
- 运算符语法触发的隐式广播失败时视为不可恢复错误，同时须提供以返回值报告错误的替代路径
- 所有错误须携带充分的诊断信息
- 不可恢复错误发生后须最小化资源泄漏风险
- 并行操作中发生不可恢复错误时须立即传播，不得静默忽略
- 所有资源释放逻辑不得引发不可恢复错误

## 28. 质量要求

### 28.1 文档

- 所有公开 API 须有文档说明
- 所有标记为不安全的函数须有安全性说明
- 关键 API 须有使用示例

### 28.2 测试

- 须包含单元测试、集成测试和边界测试

### 28.3 数值精度

- 各类运算的数值精度须满足 IEEE 754 标准
- 须保证跨平台结果一致性

### 28.4 边界覆盖

须覆盖以下边界情况：空张量、单元素、大张量、极端值、非连续布局、高维、广播形状不兼容、维度类型互转、no_std 构建验证

### 28.5 并行与 SIMD 测试

- 并行和 SIMD 相关功能须保证无数据竞争和未定义行为
- 并行归约结果须与单线程一致
