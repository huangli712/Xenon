# Xenon 需求说明书 v20

> Rust 科学计算多维数组库
> 版本: v20 | 日期: 2026-03-31 | 许可: MIT | MSRV: Rust 1.85+

---

# 第一部分：项目定位

## 1. 项目概述

Xenon 是一个 Rust 多维数组（张量）库，定位为科学计算的数值基础设施，类似于 Python 生态中 NumPy ndarray 层的多维数组基础设施。

### 1.1 目标用户

| 用户类型 | 核心诉求 |
|----------|----------|
| 库开发者 | 稳定API、零开销抽象、指针级互操作 |
| 系统开发者 | no_std支持、底层内存控制、确定性数值行为、最小依赖 |
| 间接用户 | 性能、正确性、与Python经验的直觉一致性 |

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 正确性优先 | 类型安全、内存安全、数值精度 |
| 性能敏感 | 零成本抽象、SIMD 优先路径、缓存友好的内存布局、避免不必要的分配 |
| 清晰抽象 | API 语义明确，无隐式行为 |
| 科学计算导向 | 列优先（F-order）默认、SIMD 友好、BLAS 兼容内存布局 |
| 最小依赖 | 仅允许可选的并行和SIMD相关外部依赖 |

### 1.3 工程约束

| 约束 | 要求 |
|------|------|
| Crate 结构 | 单 crate，遵循 SemVer |
| no_std 支持 | 支持 `no_std`（需 `alloc`），`std` 为默认 feature |
| Feature gates | `std`（默认）、`parallel`（并行，隐式启用 `std`）、`simd`（SIMD加速） |

---

## 2. 项目范围

### 2.1 范围内

- N 维数组的存储、构造、索引与高级索引、切片、形状操作、逐元素运算、归约、广播、集合操作、显式类型转换
- 向量内积
- 原始指针 API，供上游库零开销集成
- 自定义复数类型（FFI友好，不依赖num-complex）
- 临时工作空间（对齐分配，供上游库使用）

### 2.2 范围外

- GEMM（矩阵-矩阵乘法）、矩阵分解、对角化等高级线性代数
- FFT、稀疏矩阵、自动微分
- BLAS/LAPACK 绑定（由上游库通过指针API集成）
- GPU 后端（当前版本不实现，但须预留扩展性）
- serde 序列化、arena 分配器
- 栈分配小数组（固定大小数组建议使用 nalgebra）

# 第二部分：功能需求

## 3. 维度系统

- 须支持 0 至 6 维静态维度及动态维度（维度数在运行时确定）
- 静态维度须在编译时确定维度数，动态维度须在运行时确定
- 须提供维度类型之间的转换：静态→动态总是成功，动态→静态当维度数匹配时成功
- 须提供从常见 Rust 类型（元组、数组、切片、Vec）构造维度类型的能力
- 须提供插入轴和移除轴的能力（零维不支持移除轴，以保证编译时安全）
- 须支持负步长（用于维度反转）
- 所有涉及内存分配的路径须使用溢出检查，溢出时须返回错误
- 须为动态维度类型提供便捷的索引方式（切片、数组、Vec 等）

---

## 4. 元素类型

- 须支持以下元素类型：整数（i32、i64）、浮点（f32、f64）、复数（Complex<f32>、Complex<f64>）、bool、usize
- 须按能力分层约束类型：基础层（所有类型，值复制、相等比较、格式化、线程安全）、数值层（整数、浮点、复数，四则运算、取负、单位元、符号函数）、实数层（f32、f64，数学函数）、复数层（复数类型，复数运算）
- bool 和 usize 仅属于基础层，不支持四则运算
- 不支持自动类型提升，类型转换须显式
- 类型约束体系须封闭，下游 crate 不可为自定义类型实现

---

## 5. 复数类型

- 须提供自定义复数类型，不依赖外部 crate
- 须支持复数算术运算（四则运算、复合赋值、取负）
- 须支持复数与同精度实数的混合运算，跨精度须显式转换
- 须提供近似相等判断方法，不提供数学序关系比较
- 须支持格式化输出（调试格式和数学表示形式）
- 内存布局须与 C99 `_Complex` 兼容

---

## 6. 存储系统

- 须在编译时区分只读、可写、拥有三种访问级别
- 须支持以下存储模式：拥有数据（深拷贝）、只读借用视图（O(1) 拷贝元数据）、独占可写借用视图（不可克隆）、共享引用计数（内置写时复制）
- 共享存储的写时复制：多引用时须深拷贝，单引用时直接返回可写访问，须线程安全
- 共享存储的视图生命周期须与存储自身绑定，确保写时复制与活跃视图之间的互斥由编译器保证
- 须使用对齐的堆分配（默认 64 字节）
- 须正确处理零大小类型和空数组
- 须预留未来扩展到其他计算设备的能力（当前仅 CPU）
- 不可在公开 API 中引入 GPU 相关类型或抽象

---

## 7. 内存布局

- 默认使用列优先（F-order）布局，项目全局唯一默认
- 步长须使用有符号类型，单位为元素个数，支持负步长
- 须支持主维度填充以实现对齐，填充区域零初始化，填充字节不作为逻辑元素暴露
- 标准构造函数默认不填充，须提供创建填充布局数组的能力
- 须提供布局状态查询能力：严格连续性、宽松连续性（允许填充）、逐轴连续、SIMD 对齐、零步长检测、负步长检测、填充检测
- 须缓存布局状态以避免运行时重复计算
- 默认对齐为 64 字节，构造时可指定对齐值（须为 2 的幂，≥ 元素自然对齐）
- 小数组允许降级到元素自然对齐
- 填充量计算须使用 checked 算术

---

## 8. 张量类型

- 须提供泛型张量类型，以存储模式和维度类型为泛型参数
- 须提供常用张量类型的便捷别名（按存储模式和维度数）
- 须维护张量的元数据：各轴长度、各轴步长、数据起始偏移量、布局状态
- 须提供基础查询方法：元素数（含溢出检查变体）、形状、步长、维度数、数据指针等
- 须提供布局查询方法：连续性、对齐、零/负步长检测、填充检测等
- 所有构造路径（含从裸指针构造、视图构造等）须统一验证 shape、strides 和 offset 的合法性，防止越界访问

# 第三部分：计算需求

## 9. 计算加速

### 9.1 SIMD

- 须通过可选 feature `simd` 提供 SIMD 加速，默认关闭
- 须支持主流架构的 SIMD 指令集，运行时自动检测并分发，不支持时退化为标量路径
- 须为逐元素运算、归约、内积、比较等操作提供 SIMD 路径
- SIMD 适用的前置条件：宽松连续内存、正步长方向
- 不满足 SIMD 条件时须自动回退标量路径
- 须正确处理填充字节和尾部元素
- 整数除法和超越函数不走 SIMD 路径
- 广播操作数须提供 SIMD broadcast 优化路径
- 二元操作数布局方向不一致时须回退标量路径

### 9.2 并行

- 须通过可选 feature `parallel` 提供数据并行，默认关闭
- 并行阈值可配置，默认数据量 ≥ 256KB 时启用
- 须按连续内存块分块，分割边界须对齐以避免 false sharing
- 须支持逐元素运算、归约、map、zip 的并行执行
- 浮点归约须使用补偿求和算法保证精度，整数归约须使用 widening 累加
- 禁止嵌套并行

### 9.3 自动路径选择

- 须根据数据规模、内存布局、步长方向和 feature 启用状态自动选择最优执行路径（标量/SIMD/SIMD+并行/标量+并行）
- 二元运算须根据操作数布局兼容性、形状关系和存储复用可能性选择执行路径

---

## 10. 线程安全

- 所有存储模式须根据 Rust 的 Send/Sync 规则自动推导线程安全性
- 并行迭代须保证各线程访问不重叠的元素区间
- Padding 字节写入不得产生数据竞争

---

## 11. 迭代器

- 须支持以下迭代类型：元素迭代、轴迭代（沿指定轴产出降维子视图）、窗口迭代（滑动窗口）、索引迭代（带多维索引）、并行迭代、多数组同步迭代（zip）
- 须支持指定遍历顺序（F/C）
- 所有迭代器须提供精确的元素数量信息
- 连续数组的元素迭代器须支持双向遍历
- 空数组迭代须立即结束
- 填充数组的迭代须仅遍历逻辑元素
- 禁止对广播视图进行可变迭代
- 窗口迭代须支持非连续源数组
- 零维张量元素迭代产出恰好 1 个元素，不支持轴迭代和窗口迭代

---

# 第四部分：API 需求

## 12. 逐元素运算

- 须支持算术运算（加、减、乘、除），适用于数值类型（整数、浮点、复数）
- 须支持一元运算：绝对值、取反、符号函数、平方
- 须支持数学函数（仅实数类型 f32/f64）：正弦、平方根、指数、自然对数、下取整、上取整
- 须支持复数运算：模（返回实数类型张量）、共轭
- 须支持逻辑非（仅 bool）
- 须支持比较运算（等于、不等于、小于、大于），返回 bool 类型张量，NaN 比较遵循 IEEE 754
- 须支持标量与张量的逐元素运算（标量视为 0D 张量广播）
- 所有二元运算和比较运算须支持广播

## 13. 矩阵运算

- 须支持向量内积（dot）
- 复数内积须使用 Hermitian 内积（与 BLAS cdotc/zdotc 一致）
- 维度或形状不匹配时须提供可恢复的错误处理路径

## 14. 归约

- 须支持 sum 归约
- 须支持全局归约和沿指定轴归约
- 须提供 keepdims 变体（保留归约轴为 size-1）
- 整数归约溢出时须 panic（debug 和 release 模式一致）
- 空数组 sum 归约返回加法单位元

## 15. 集合操作

- 须提供唯一值操作（unique），返回排序后的不重复元素
- 须为浮点、复数定义全序比较规则（浮点使用 IEEE 754 全序语义，复数使用字典序，NaN 视为相等并排在末尾）
- 须支持按自定义键去重
- bool 和 usize 不适用

## 16. 广播

- 须遵循 NumPy 广播规则（右对齐，size-1 维度拉伸）
- 广播为零拷贝操作
- 广播产生的视图始终为只读
- 须提供显式广播方法，返回零拷贝广播视图
- 须提供可恢复的错误处理路径（广播失败时返回错误而非 panic）
- 原地运算的左操作数不可被广播，须为连续内存
- 广播后维度数 > 6 时须使用动态维度类型

## 17. 形状操作

- 须提供转置操作（全轴反转和指定轴排列），始终零拷贝
- 须提供 reshape 操作，不改变维度类型（静态保持静态，动态保持动态）

## 18. 索引操作

- 须支持多维整数索引、范围索引（切片）、布尔掩码高级索引
- 布尔掩码索引须返回一维数组，按 F-order 遍历顺序收集，始终产生数据拷贝

## 19. 构造操作

- 须提供 zeros、ones、eye 等标准构造方法
- 须支持从 Vec、切片、数组、标量值构造张量
- 嵌套集合构造须验证内层一致性，不一致时返回错误

## 20. 运算符重载

- 须为四则运算符（+、-、*、/）提供重载，隐式支持广播
- 须支持张量与张量、张量与标量之间的运算
- 按值传入的操作数允许存储复用（连续且形状一致时），按引用传入的不复用
- 所有组合均返回 Owned 张量
- 当前版本不支持比较、复合赋值、一元、位运算符的重载

## 21. 实用操作

- 须提供 clip（裁剪到 [min, max] 范围，NaN 保持不变，min > max 时 panic）
- 须提供 fill（填充所有逻辑元素，广播视图时 panic）

## 22. 连续性保证

- 须提供连续性保证操作：引用输入（拷贝）、零拷贝视图（仅当已满足时）、消耗所有权（按需拷贝）
- 须同时提供宽松连续性变体（允许填充张量零拷贝）
- 连续性判定使用严格连续性，填充数组和广播视图不满足严格连续性

## 23. 类型转换

- 须提供逐元素类型转换方法（cast），溢出使用饱和语义
- 须同时提供同类型拷贝方法，用于泛型管线
- 仅 Owned 存储模式支持 cast
- 须支持各存储模式之间的转换
- 须支持静态维度与动态维度之间的互转
- 须为张量类型实现标准库转换 trait
- 不支持跨类别隐式转换
- 须为所有支持的类型转换组合定义明确的转换语义

## 24. 格式化输出

- Display 须面向用户，简洁可读，不显示内部实现细节
- 数据展示须遵循 NumPy 风格
- 须为不同维度（0D、1D、2D、3D+、空数组）定义明确的输出格式
- 大数组须触发截断，保留首尾部分数据

## 25. FFI 集成

- 须提供原始指针 API，供上游库零开销集成（含 unchecked 变体）
- 须支持从裸指针构造张量和将张量解构为裸指针（unsafe）
- 须提供字节单位的步长和偏移量查询
- 须提供 BLAS 兼容性查询与布局转换 API（leading dimension、布局兼容性检查、布局转换）
- 须提供多维索引到指针和偏移量的转换（含 checked 和 unchecked 变体）
- 须提供转置标识枚举（不转置、转置、共轭转置）

## 26. 临时工作空间

- 须提供临时工作空间类型，支持对齐分配（默认 64 字节）
- 须支持 no_std + alloc 环境
- 须支持子缓冲区分割、递归分割、自动扩容（保持对齐）
- 须支持多种借出方式（按字节、按元素类型、按需初始化、源数据拷贝写入）
- 活跃借用期间不允许扩容
- 须实现 Send，不实现 Sync

# 第五部分：工程保障

## 27. 错误处理

- 可恢复错误（形状不兼容、广播失败、布局不满足、轴索引无效、构造参数无效、维度互转失败、参数语义不合法等）须返回 Result
- 不可恢复错误（索引越界、内存分配失败、前置条件违反等）须 panic
- 索引越界须同时提供 unchecked 变体
- 运算符重载触发的隐式广播失败时 panic，同时须提供返回 Result 的替代路径
- 所有错误须携带充分的诊断信息
- 错误类型须兼容 no_std 环境
- panic 后须保证可安全 drop（无 UB、无内存泄漏），不保证数据一致性
- 并行操作 panic 时须立即传播，不得吞没
- 所有 drop 实现不得 panic

## 28. 质量要求

### 28.1 文档

- 所有公开 API 须有 doc comment
- 所有 unsafe 函数须有 Safety 文档节
- 关键 API 须有使用示例

### 28.2 测试

- 须包含单元测试、集成测试和边界测试
- 行覆盖率 ≥ 90%，unsafe 代码块覆盖率 ≥ 95%，关键路径分支覆盖率 ≥ 85%

### 28.3 数值精度

- 各类运算须满足对应的精度要求（ULP 或相对误差）
- 整数归约溢出时须 panic（debug 和 release 模式一致）
- 须覆盖边界值测试（0、±1、极大值、极小正规数、subnormal、±Inf、NaN 等）
- 须提供固定种子的回归测试（golden test）
- 须保证跨平台结果一致性

### 28.4 边界覆盖

须覆盖以下边界情况：空张量、单元素、大张量、极端值（NaN/Inf/subnormal）、非连续布局、高维（≥4维）、广播形状不兼容、写时复制行为差异、对齐降级、填充布局步长正确性、维度类型互转（匹配/不匹配）、no_std 构建验证

### 28.5 并行与 SIMD 测试

- CI 中须包含数据竞争检测和未定义行为检测
- SIMD 路径结果须与标量路径在精度容差内一致
- 并行归约结果须与单线程一致（浮点允许精度容差内差异）
- 须测试不同 SIMD 指令集上结果的一致性
- 须测试元素数在并行阈值附近的边界情况

### 28.6 CI 要求

- 须覆盖主流平台（Linux、macOS、Windows）及主流架构
- 须覆盖 stable、MSRV 和 nightly 版本
- 须覆盖默认 features、无默认 features、全 features 组合
- 零格式问题和零 lint warning
- 须提供性能基准测试，记录历史结果
