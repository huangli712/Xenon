# Xenon 需求说明书 v21

> Rust 科学计算多维数组库
> 版本: v21 | 日期: 2026-04-06 | 许可: MIT | MSRV: Rust 1.85+

---

# 第一部分：项目定位

## 1. 项目概述

Xenon 是一个 Rust 多维数组（张量）库，定位为科学计算的数值基础设施，类似于 Python 生态中 NumPy ndarray 层的多维数组基础设施。

### 1.1 目标用户

| 用户类型 | 核心诉求 |
|----------|----------|
| 库开发者 | 稳定API、零开销抽象、指针级互操作 |
| 系统开发者 | no_std支持、底层内存控制、确定性数值行为、最小依赖 |
| 间接用户 | 性能、正确性、与Python经验的直觉一致性 |

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 正确性优先 | 类型安全、内存安全、数值精度 |
| 性能敏感 | 零成本抽象、缓存友好的内存布局 |
| 清晰抽象 | API 语义明确，无隐式行为 |
| 科学计算导向 | BLAS 兼容 |
| 最小依赖 | 仅允许可选的并行和SIMD相关外部依赖 |

### 1.3 工程约束

| 约束 | 要求 |
|------|------|
| Crate 结构 | 单 crate，遵循 SemVer |
| no_std 支持 | 支持 `no_std`，`std` 为默认 |
| Feature gates | 须提供可选的并行和 SIMD 加速功能 |

---

## 2. 项目范围

### 2.1 范围内

- N 维数组的存储、构造、索引与高级索引、切片、形状操作、逐元素运算、归约、广播、集合操作、显式类型转换
- 向量内积
- 原始指针 API，供上游库零开销集成
- 自定义复数类型（FFI友好，不依赖num-complex）
- 临时工作空间（对齐分配，供上游库使用）

### 2.2 范围外

- GEMM（矩阵-矩阵乘法）、矩阵分解、对角化等高级线性代数
- FFT、稀疏矩阵、自动微分
- BLAS/LAPACK 绑定（由上游库通过指针API集成）
- GPU 后端（当前版本不实现，但须预留扩展性）
- serde 序列化、arena 分配器
- 栈分配小数组（固定大小数组建议使用 nalgebra）

# 第二部分：功能需求

## 3. 维度系统

- 须支持 0 至 6 维静态维度及动态维度（维度数在运行时确定）
- 须提供维度类型之间的转换：静态→动态总是成功，动态→静态当维度数匹配时成功
- 须提供从常见 Rust 类型构造维度类型的能力
- 须提供插入轴和移除轴的能力（零维不支持移除轴）
- 须支持负步长（用于维度反转）
- 所有涉及内存分配的路径须进行溢出检查，溢出时须返回错误

---

## 4. 元素类型

- 须支持以下元素类型：整数（i32、i64）、浮点（f32、f64）、复数（Complex<f32>、Complex<f64>）、bool、usize
- 须按运算能力对类型进行分层约束：基础操作（所有类型）、四则运算（整数、浮点、复数）、数学函数（仅实数）、复数运算（仅复数）
- bool 和 usize 不支持四则运算
- 不支持自动类型提升，类型转换须显式
- 类型约束体系须封闭，下游 crate 不可为自定义类型实现

---

## 5. 复数类型

- 须提供自定义复数类型，不依赖外部 crate
- 须支持复数算术运算（四则运算、复合赋值、取负）
- 须支持复数与同精度实数的混合运算，跨精度须显式转换
- 须提供近似相等判断方法，不提供数学序关系比较
- 须支持格式化输出（调试格式和数学表示形式）
- 内存布局须与 C99 `_Complex` 兼容

---

## 6. 存储系统

- 须在编译时区分只读、可写、拥有三种访问级别
- 须支持以下存储模式：拥有数据、只读借用视图、独占可写借用视图、共享引用计数
- 共享存储须线程安全
- 须正确处理零大小类型和空数组
- 须预留未来扩展到其他计算设备的能力
- 不可在公开 API 中引入 GPU 相关类型或抽象

---

## 7. 内存布局

- 默认使用列优先（F-order）布局
- 步长须支持负步长
- 须支持主维度填充以实现对齐，填充区域零初始化，填充字节不作为逻辑元素暴露
- 标准构造函数默认不填充，须提供创建填充布局数组的能力
- 须提供布局状态查询能力：连续性、对齐、零步长、负步长、填充等

---

## 8. 张量类型

- 须提供泛型张量类型，以存储模式和维度类型为泛型参数
- 须提供常用张量类型的便捷别名
- 须维护张量的元数据：各轴长度、各轴步长、数据起始偏移量、布局状态
- 须提供基础查询方法：元素数、形状、步长、维度数、数据指针等
- 须提供布局查询方法：连续性、对齐、零/负步长、填充等
- 所有构造路径须统一验证合法性，防止越界访问

---

# 第三部分：计算需求

## 9. 计算加速

### 9.1 SIMD

- 须通过可选 feature 提供 SIMD 加速，默认关闭
- 须为逐元素运算、归约、内积、比较等操作提供加速路径
- 不满足 SIMD 条件时须自动回退标量路径
- 须正确处理填充字节和尾部元素

### 9.2 并行

- 须通过可选 feature 提供数据并行，默认关闭
- 并行阈值可配置
- 须支持逐元素运算、归约、map、zip 的并行执行
- 浮点归约须保证精度
- 禁止嵌套并行

### 9.3 自动路径选择

- 须根据数据规模、内存布局和 feature 启用状态自动选择最优执行路径

---

## 10. 线程安全

- 所有存储模式须遵循 Rust 的 Send/Sync 规则
- 并行操作须保证线程安全

---

## 11. 迭代器

- 须支持以下迭代类型：元素迭代、轴迭代、窗口迭代、索引迭代、并行迭代、多数组同步迭代（zip）
- 须支持指定遍历顺序（F/C）
- 所有迭代器须提供精确的元素数量信息
- 空数组迭代须立即结束
- 填充数组的迭代须仅遍历逻辑元素
- 禁止对广播视图进行可变迭代
- 零维张量元素迭代产出恰好 1 个元素，不支持轴迭代和窗口迭代

# 第四部分：API 需求

## 12. 逐元素运算

- 须支持算术运算（加、减、乘、除），适用于数值类型（整数、浮点、复数）
- 须支持一元运算：绝对值、取反、符号函数、平方
- 须支持数学函数（仅实数类型 f32/f64）：正弦、平方根、指数、自然对数、下取整、上取整
- 须支持复数运算：模（返回实数类型张量）、共轭
- 须支持逻辑非（仅 bool）
- 须支持比较运算（等于、不等于、小于、大于），返回 bool 类型张量，NaN 比较遵循 IEEE 754
- 须支持标量与张量的逐元素运算
- 所有二元运算和比较运算须支持广播

## 13. 矩阵运算

- 须支持向量内积（dot）
- 复数内积须使用 Hermitian 内积
- 维度或形状不匹配时须提供可恢复的错误处理路径

## 14. 归约

- 须支持 sum 归约
- 须支持全局归约和沿指定轴归约
- 须提供 keepdims 变体（保留归约轴为 size-1）
- 整数归约溢出时须 panic
- 空数组 sum 归约返回加法单位元

## 15. 集合操作

- 须提供唯一值操作（unique），返回排序后的不重复元素
- 须为浮点、复数定义全序比较规则
- 须支持按自定义键去重
- bool 和 usize 不适用

## 16. 广播

- 须遵循 NumPy 广播规则
- 广播为零拷贝操作
- 广播产生的视图始终为只读
- 须提供显式广播方法
- 须提供可恢复的错误处理路径

## 17. 形状操作

- 须提供转置操作，始终零拷贝
- 须提供 reshape 操作

## 18. 索引操作

- 须支持多维整数索引、范围索引（切片）、布尔掩码高级索引
- 布尔掩码索引须返回一维数组，始终产生数据拷贝

## 19. 构造操作

- 须提供 zeros、ones、eye 等标准构造方法
- 须支持从 Vec、切片、数组、标量值构造张量
- 嵌套集合构造须验证内层一致性

## 20. 运算符重载

- 须为四则运算符（+、-、*、/）提供重载，隐式支持广播
- 须支持张量与张量、张量与标量之间的运算
- 所有组合均返回 Owned 张量
- 当前版本不支持比较、复合赋值、一元、位运算符的重载

## 21. 实用操作

- 须提供 clip（裁剪到 [min, max] 范围，NaN 保持不变，min > max 时 panic）
- 须提供 fill（填充所有逻辑元素，广播视图时 panic）

## 22. 连续性保证

- 须提供连续性保证操作
- 须同时提供宽松连续性变体

## 23. 类型转换

- 须提供逐元素类型转换方法（cast），溢出行为须明确定义
- 须同时提供同类型拷贝方法
- 仅 Owned 存储模式支持 cast
- 须支持各存储模式之间的转换
- 须支持静态维度与动态维度之间的互转
- 须为张量类型实现标准库转换 trait
- 不支持跨类别隐式转换

## 24. 格式化输出

- Display 须面向用户，简洁可读
- 数据展示须遵循 NumPy 风格
- 大数组须触发截断

## 25. FFI 集成

- 须提供原始指针 API，供上游库零开销集成（含 unchecked 变体）
- 须支持从裸指针构造张量和将张量解构为裸指针（unsafe）
- 须提供 BLAS 兼容性相关 API
- 须提供多维索引到指针和偏移量的转换（含 checked 和 unchecked 变体）
- 须提供转置标识枚举

## 26. 临时工作空间

- 须提供临时工作空间类型，支持对齐分配
- 须支持 no_std 环境
- 须支持子缓冲区分割、递归分割、自动扩容
- 活跃借用期间不允许扩容

# 第五部分：工程保障

## 27. 错误处理

- 可恢复错误须返回 Result
- 不可恢复错误须 panic
- 索引越界须同时提供 unchecked 变体
- 运算符重载触发的隐式广播失败时 panic，同时须提供返回 Result 的替代路径
- 所有错误须携带充分的诊断信息
- 错误类型须兼容 no_std 环境
- panic 后须保证可安全 drop（无 UB、无内存泄漏）
- 并行操作 panic 时须立即传播，不得吞没
- 所有 drop 实现不得 panic

## 28. 质量要求

### 28.1 文档

- 所有公开 API 须有 doc comment
- 所有 unsafe 函数须有 Safety 文档节
- 关键 API 须有使用示例

### 28.2 测试

- 须包含单元测试、集成测试和边界测试
- 须达到较高的代码覆盖率

### 28.3 数值精度

- 各类运算须满足对应的精度要求
- 须覆盖边界值测试
- 须保证跨平台结果一致性

### 28.4 边界覆盖

须覆盖以下边界情况：空张量、单元素、大张量、极端值、非连续布局、高维、广播形状不兼容、维度类型互转、no_std 构建验证

### 28.5 并行与 SIMD 测试

- 须包含数据竞争检测和未定义行为检测
- SIMD 路径结果须与标量路径一致
- 并行归约结果须与单线程一致

### 28.6 CI 要求

- 须覆盖主流平台及主流架构
- 须覆盖多个 Rust 版本
- 须覆盖不同 feature 组合
