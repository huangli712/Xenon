# 第一部分：项目定位

## 1. 项目概述

Xenon 是一个 Rust 多维数组（张量）库，定位为科学计算的数值基础设施，类似于 Python 生态中 NumPy ndarray 层的多维数组基础设施。

### 1.1 目标用户

| 用户类型 | 核心诉求 |
|----------|----------|
| 库开发者 | 稳定API、零开销抽象、指针级互操作 |
| 系统开发者 | no_std支持、底层内存控制、确定性数值行为、最小依赖 |
| 间接用户 | 性能、正确性、与Python经验的直觉一致性 |

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 正确性优先 | 类型安全、内存安全、数值精度 |
| 性能敏感 | 零成本抽象、SIMD 优先路径、缓存友好的内存布局、避免不必要的分配 |
| 清晰抽象 | API 语义明确，无隐式行为 |
| 科学计算导向 | 列优先（F-order）默认、SIMD 友好、BLAS 兼容内存布局 |
| 最小依赖 | 仅 rayon（可选）和 pulp（可选）。`InlineArray` 为库内自研类型（无外部依赖），用于 IxDyn 动态维度和广播形状推导的栈分配优化（≤ 8 维内联，> 8 维回退堆分配） |

### 1.3 工程约束

| 约束 | 要求 |
|------|------|
| Crate 结构 | 单 crate，遵循 SemVer |
| no_std 支持 | 支持 `no_std`（需 `alloc`），`std` 为默认 feature|
| Feature gates | `std`（默认）、`parallel`（rayon，隐式启用 `std`，因 rayon 依赖线程）、`simd`（pulp） |

---

## 2. 项目范围

### 2.1 范围内

- N 维数组的存储、构造、索引与高级索引、切片、形状操作、逐元素运算、归约、广播、集合操作、显式类型转换
- 向量内积
- 原始指针 API，供上游库零开销集成
- 自定义复数类型（FFI友好，不依赖num-complex）
- 临时工作空间（对齐分配，供上游库使用）

### 2.2 范围外

- GEMM（矩阵-矩阵乘法）、矩阵分解、对角化等高级线性代数
- FFT、稀疏矩阵、自动微分
- BLAS/LAPACK 绑定（由上游库通过指针API集成）
- GPU 后端（当前版本不实现，但须预留扩展性）
- serde 序列化、arena 分配器
- 栈分配小数组（固定大小数组建议使用 nalgebra）

# 第二部分：核心架构

## 3. 维度系统

### 3.1 维度类型总览

支持 0 至 6 维静态维度 + 动态维度。所有维度类型须实现 `Dimension` trait（见 §3.2），并满足 `Debug + Clone + Eq + Hash + Send + Sync`。

| 类型 | 含义 | 构造方式 |
|------|------|----------|
| Ix0 | 0 维（标量） | `Ix0`（无参，实现 `Default`） |
| Ix1 | 1 维（向量） | `Ix1(usize)` 或 `From<usize>` |
| Ix2 | 2 维（矩阵） | `Ix2(usize, usize)` 或 `From<(usize, usize)>` |
| Ix3 | 3 维 | `Ix3(usize, usize, usize)` 或 `From<(usize, usize, usize)>` |
| Ix4 | 4 维 | `From<[usize; 4]>` / `From<(usize, usize, usize, usize)>` |
| Ix5 | 5 维 | `From<[usize; 5]>` / `From<(usize, usize, usize, usize, usize)>` |
| Ix6 | 6 维 | `From<[usize; 6]>` / `From<(usize, usize, usize, usize, usize, usize)>` |
| IxDyn | 动态维度 | `From<&[usize]>` / `From<Vec<usize>>` |

IxDyn 须使用动态长度的内部存储。库须提供一个 `pub` 的小型内联数组类型（无外部依赖），用于 IxDyn 内部存储、广播形状推导（§16.1）、错误类型中的 ShapeData（§27）等需要动态长度但通常维度较低的场景。

### 3.2 Dimension trait

所有维度类型须实现 `Dimension` trait。下游 crate 不可为自定义类型实现。

| 关联类型/方法 | 签名 | 说明 |
|--------------|------|------|
| `type Pattern` | 关联类型 | 索引模式类型，用于 `Index` trait 实现 |
| `type Larger` | 关联类型：`Dimension` | 插入一个轴后的维度类型 |
| `NDIM` | 关联常量：`Option<usize>` | 编译时维度数。静态维度返回 `Some(N)`，IxDyn 返回 `None` |
| `ndim()` | `&self -> usize` | 返回维度数 |
| `slice()` | `&self -> &[usize]` | 返回 shape 切片 |
| `slice_mut()` | `&mut self -> &mut [usize]` | 返回可变 shape 切片（仅库内可用） |
| `size()` | `&self -> usize` | 返回总元素数（各轴乘积）。Ix0 返回 1。仅用于非分配场景 |
| `size_checked()` | `&self -> Option<usize>` | 返回总元素数，溢出时返回 `None`。Ix0 返回 `Some(1)`。所有涉及内存分配的路径须使用此方法 |
| `size_wrapping()` | `&self -> usize` | 返回总元素数，溢出时 wrapping |
| `insert_axis()` | `&self, axis: usize -> Self::Larger` | 在指定位置插入长度为 1 的新轴。axis 须在 `[0, self.ndim()]` 范围内，否则 panic |
| `default_strides()` | `&self -> Self` | 计算 F-order 默认步长（列优先） |
| `into_dyn()` | `self -> IxDyn` | 转换为动态维度，消费 self。静态→动态为 infallible；反向使用 `TryFrom<IxDyn>`（见 §3.5） |

**步长存储**：shape 与 strides 共用同一维度类型 `D`，步长须支持负值（反转维度），所有步长的绝对值须 ≤ `isize::MAX as usize`。

**分配安全**：所有涉及内存分配的路径须使用 `size_checked()`。当 `size_checked()` 返回 `None` 时，须返回 `ShapeError::Overflow` 错误（见 §27），不得 panic 或静默继续。

**IxDyn 索引发捷性**：须为 `TensorBase<S, IxDyn>` 额外实现以下 `Index` 变体：

| 索引类型 | 实现方式 |
|----------|----------|
| `&[usize]` | `Index<&[usize]> for TensorBase<S, IxDyn>` |
| `[usize; N]` | `Index<[usize; N]> for TensorBase<S, IxDyn>`（const generic） |
| `Vec<usize>` | `Index<Vec<usize>> for TensorBase<S, IxDyn>` |

### 3.3 RemoveAxis trait

`RemoveAxis` 是独立的 trait，仅由可降维的维度类型实现。**Ix0 不实现此 trait**，从编译时保证零维张量不会调用 `remove_axis`。

| 关联类型/方法 | 签名 | 说明 |
|--------------|------|------|
| `type Smaller` | 关联类型：`Dimension` | 移除一个轴后的维度类型 |
| `remove_axis()` | `&self, axis: usize -> Self::Smaller` | 移除指定轴，返回降维后的维度 |

Ix1~Ix6 和 IxDyn 实现 `RemoveAxis`。

### 3.4 IntoDimension trait

`IntoDimension` 是泛型转换 trait，将各种输入类型统一转换为 `Dimension` 实现者。

| 关联类型/方法 | 签名 | 说明 |
|--------------|------|------|
| `type Dim` | 关联类型：`Dimension` | 转换目标维度类型 |
| `into_dimension()` | `self -> Self::Dim` | 消费 self，返回目标维度类型实例 |

须为以下输入类型提供实现：`()` → Ix0、`usize` → Ix1、二元组 → Ix2、三元组 → Ix3、四元组 → Ix4、五元组 → Ix5、六元组 → Ix6、`[usize; N]`（N = 0~6）→ IxN、`Vec<usize>` → IxDyn、`IxDyn` → IxDyn、所有 `Dimension` 实现者 → 自身。

### 3.5 维度互转规则

**静态 → 动态（IxN → IxDyn）**：总是成功，保留 shape 信息。

**动态 → 静态（IxDyn → IxN）**：当维度数匹配时成功，否则返回 `DimensionMismatch`。

| 转换方向 | API | 说明 |
|----------|-----|------|
| 静态 → 动态 | `dim.into_dyn()`（`Dimension` trait 方法） | Infallible，消费 self |
| 动态 → 静态 | `IxN::try_from(dyn_dim)`（标准库 `TryFrom`） | Fallible，返回 `Result<IxN, DimensionMismatch>` |

`IntoDimension` trait（§3.4）仅处理 identity 转换（如 `usize → Ix1`），不负责跨类型维度转换。

### 3.6 边界情况与设计约束

**Ix0 边界语义**

| 属性 | 值 |
|------|-----|
| shape | `&[]`（空切片） |
| ndim | 0 |
| 元素数 | 1 |
| 与 IxDyn 互转 | IxDyn.ndim() == 0 时可双向转换 |

**default_strides 边界行为**

| 边界情况 | 行为 |
|----------|------|
| Ix0 | 返回 Ix0（空，无步长可计算） |
| shape 含零维度 | 对应轴步长值不影响实际寻址 |

**reshape 与维度类型约束**

- reshape 不改变维度类型（静态保持静态，动态保持动态）
- 显式转换维度类型使用 `into_dyn()` 或 `IxN::try_from()`
- 静态维度 reshape 时目标维度数须匹配类型
- Ix6 不可 `unsqueeze`（已达最大静态维度）。需要插入轴时，须先转换为 `IxDyn`

## 4. 元素类型体系

### 4.1 类型体系

四层元素类型体系，每层在前一层能力基础上增加新能力：

| 层次 | 适用类型 | 能力 |
|------|----------|------|
| 基础层 | 整数、浮点、复数、bool、usize | 值复制、默认值、相等比较、格式化输出、线程安全 |
| 数值层 | 整数、浮点、复数（不含 bool、usize） | 在基础层之上支持四则运算、取负、加法/乘法单位元、符号函数 |
| 实数层 | f32, f64 | 在数值层之上提供数学函数（绝对值、平方根、对数、指数、三角函数、取整）和特殊值检测 |
| 复数层 | Complex\<f32\>, Complex\<f64\> | 在数值层之上提供复数运算（实虚部访问、共轭、模、指数、对数、平方根、极坐标构造） |

**约束**：bool 和 usize 仅属于基础层，不支持四则运算。不支持自动类型提升，类型转换须显式。

### 4.2 封闭性要求

四层 trait 须封闭，下游 crate 不可为自定义类型实现。仅库内预定义类型可作为实现者：i32、i64、f32、f64、bool、usize、Complex\<f32\>、Complex\<f64\>。其中 bool 和 usize 仅属于基础层。

### 4.3 基础层（Element）

| 能力 | 说明 |
|------|------|
| 值复制 | 可按位复制 |
| 默认值 | 整数: 0, 浮点: 0.0, bool: false, 复数: (0, 0), usize: 0 |
| 相等比较 | 逐值相等 |
| 格式化 | 支持调试和用户友好的格式化输出 |
| 线程安全 | 可跨线程传递和共享 |

### 4.4 数值层（Numeric）

在基础层之上，额外要求：

| 能力 | 说明 |
|------|------|
| zero() | 加法单位元（整数: 0, 浮点: 0.0, 复数: (0, 0)） |
| one() | 乘法单位元（整数: 1, 浮点: 1.0, 复数: (1, 0)） |
| signum() | 符号函数 |
| 四则运算 | 加、减、乘、除 |
| 取负 | 一元负号运算 |

### 4.5 实数层（RealScalar）

在数值层之上，额外提供：

**数学函数**

| 方法 | 说明 |
|------|------|
| abs() | 绝对值 |
| sqrt() | 平方根 |
| ln() | 自然对数 |
| exp() | 指数函数 |
| sin() | 正弦 |
| floor(), ceil() | 取整 |

### 4.6 复数层（ComplexScalar）

在数值层之上，额外提供。

**关联类型**：

| 关联类型 | 说明 |
|----------|------|
| Real | 底层实数类型 |

**方法与常量**：

| 方法/常量 | 说明 |
|-----------|------|
| re() | 实部 |
| im() | 虚部 |
| conj() | 共轭 |
| norm() | 模 |
| exp() | 复数指数 |
| ln() | 复数对数 |
| sqrt() | 复数平方根（主值） |
| i() | 虚数单位 |

### 4.7 NaN/Inf 处理约定

| 场景 | 行为 |
|------|------|
| 归约（sum）含 NaN | 结果为 NaN |
| 排序/比较含 NaN | NaN 不参与排序，偏序比较返回 None |
| Inf 参与算术 | 遵循 IEEE 754 规则 |
| 0.0 / 0.0 | 返回 NaN |
| 1.0 / 0.0 | 返回 Inf |

## 5. 复数类型

### 5.1 类型定义

- 自定义 Complex<T> 类型，`#[repr(C)]` 布局，不依赖外部 crate
- 实现 Copy + Clone（当 T: Copy 时），值语义
- Default 值为 Complex { re: T::default(), im: T::default() }，即 Complex(0, 0)
- Numeric trait的 zero() → Complex(0, 0)，one() → Complex(1, 0)

### 5.2 格式化输出

| trait | 格式 | 示例 |
|-------|------|------|
| Debug | 显示 re 和 im 字段 | `Complex { re: 1.0, im: 2.0 }` |
| Display | 数学表示 `{re}+{im}i`，虚部为负时符号为 `-` | `"1+2i"`, `"1-2i"`, `"0+1i"`, `"3+0i"` |

### 5.3 类型转换

| 转换 | 行为 |
|------|------|
| `From<T> for Complex<T>` | 其中 T: Element，构造 Complex(re, 0) |
| `From<(T, T)> for Complex<T>` | 从元组构造 Complex(t.0, t.1) |
| `From<Complex<T>> for (T, T)` | 反向转换，返回 (re, im) |
| 实数 → 复数 | 须使用 `Complex::from(value)` 或 `Complex::new(value, 0)`（参见 §23.1） |
| 复数 → 实数 | 不允许隐式转换，须使用 `.re` 显式取实部 |

### 5.4 算术运算

**复数与复数运算**

| 操作 | 行为 |
|------|------|
| Complex + Complex | 逐分量加 |
| Complex - Complex | 逐分量减 |
| Complex * Complex | 标准复数乘法，遵循 IEEE 754 逐分量行为 |
| Complex / Complex | 复数除法，须采用数值稳定的算法避免中间结果溢出。分母为零时返回 NaN + NaN\*i。需 `T: RealScalar` |

**复数与实数互操作**

| 操作 | 行为 |
|------|------|
| Complex + 实数 | 实数隐式提升为 Complex(r, 0.0)，结果为 Complex。仅限同精度（Complex\<f64\> + f64） |
| 实数 + Complex | 同上，交换律成立 |
| Complex * 实数 | 标量乘法。仅限同精度 |
| Complex / 实数 | 标量除法。仅限同精度 |
| 实数 / Complex | 实数除以复数。分母为零时遵循 IEEE 754。仅限同精度 |
| Complex 与整数 | 不支持隐式互操作，须先将整数转为浮点 |
| 跨精度（如 f32 + Complex\<f64\>） | 不支持隐式互操作，须先显式 cast 到同精度 |

**复合赋值与一元运算**

| 操作 | 行为 |
|------|------|
| Complex += Complex | 逐分量加后赋值 |
| Complex += 实数 | 同精度实数加后赋值 |
| Complex -= Complex | 逐分量减后赋值 |
| Complex -= 实数 | 同精度实数减后赋值 |
| Complex \*= Complex | 复数乘法后赋值 |
| Complex \*= 实数 | 同精度标量乘后赋值 |
| Complex /= Complex | 复数除法后赋值 |
| Complex /= 实数 | 同精度标量除后赋值 |
| -Complex | 一元取负 |

### 5.5 相等与比较

| 属性 | 要求 |
|------|------|
| PartialEq | 逐分量比较（re == re && im == im），NaN != NaN |
| Eq | 不实现（因 NaN 破坏自反性） |
| PartialOrd | 不实现（复数无自然全序） |
| Ord | 不实现 |
| Hash | 不实现（因无 Eq，NaN 使 hash 语义不一致） |
| 近似相等 | 提供 `approx_eq(&self, other: &Self, epsilon: T) -> bool` 方法，逐分量判断各分量差值的绝对值不超过 epsilon |
| 全序比较（工具方法） | 提供 `total_cmp(&self, other: &Self) -> Ordering` 方法，**不作为数学序关系**，仅供集合操作（如 §15.2 unique）内部排序使用 |

**设计说明**：`total_cmp` 不实现为 `Ord` trait，因为字典序不反映复数的数学性质。

### 5.6 内存布局

| 属性 | 要求 |
|------|------|
| 内存布局 | `#[repr(C)]`，保证 [re, im] 连续排列 |
| 大小 | size_of::<Complex<T>>() == 2 * size_of::<T>() |
| 对齐 | align_of::<Complex<T>>() == align_of::<T>() |
| 与 C 互操作 | 内存布局兼容 C99 `_Complex`。FFI 仅通过指针传递安全（`*const`/`*mut`），不保证按值传递的 ABI 兼容性 |
| 数组布局 | Complex<f64> 数组与交错实虚 f64 数组内存等价 |
| SIMD 对齐 | Complex<f64> 数组建议 16 字节对齐 |
| no_std 注意 | `norm()`、`exp()`、`ln()`、`sqrt()`、`from_polar()` 依赖浮点数学函数。`no_std` 环境下需通过外部数学库提供这些函数。若目标平台无浮点单元，`Complex` 的运算不可用，须在文档中明确标注 |
| Feature gate 分级（v2 路线图） | 将依赖外部数学函数的方法按 feature gate 分层，使 `no_std` 环境可按需启用。基础运算（构造、四则、共轭）不依赖外部库；数学函数方法（`norm()`/`exp()`/`ln()`/`sqrt()`）通过可选 feature 启用，`no_std` 用户可精确控制依赖。默认配置下所有方法可用，不影响现有用户 |

## 6. 存储系统

### 6.1 分层 Storage trait

存储须区分三种访问级别：**只读**、**可写**、**拥有**（编译时保证）。采用分层 trait 设计：所有存储模式实现 `Storage`（只读），可写存储额外实现 `StorageMut`（可写）。编译器须在泛型约束处自动拒绝只读存储的可写访问。

**`Storage` trait**（所有存储模式实现 — 只读访问）：

| 关联类型/方法 | 签名 | 说明 |
|--------------|------|------|
| `type Elem` | 关联类型 | 元素类型 |
| `type Device` | 关联类型：`Device` | 存储所在的计算设备（当前仅 `Cpu`） |
| `capacity()` | `&self -> usize` | 返回存储的缓冲区容量（含 padding）。此值 ≥ 逻辑元素数（`TensorBase::len()`） |
| `as_ptr()` | `&self -> *const Self::Elem` | 底层数据指针（只读） |
| `manages_memory()` | `&self -> bool` | 存储是否管理内存生命周期 |

**`StorageMut` trait**（仅可写存储实现 — 在 `Storage` 之上添加可写能力）：

| 方法 | 签名 | 说明 |
|------|------|------|
| `as_mut_ptr()` | `&mut self -> *mut Self::Elem` | 底层数据指针（可写） |

**约束**：

- `Storage` 不包含可写指针方法，确保只读存储在编译时无法获得可写指针
- `StorageMut: Storage`，子 trait 反映可写必然可读的语义
- ArcRepr 不实现 `StorageMut`，可写访问须通过 `make_mut()` 方法（见 §6.3）

### 6.2 四种存储模式

| 存储模式 | 拥有数据 | 可读 | 可写 | 克隆语义 | 典型用途 |
|----------|---------|------|------|----------|----------|
| Owned | 是 | 是 | 是 | 深拷贝 | 数组创建、运算结果 |
| ViewRepr | 否（借用） | 是 | 否 | O(1) 拷贝视图元数据 | 切片、子数组只读访问 |
| ViewMutRepr | 否（独占借用） | 是 | 是 | 不可克隆 | 原地修改子区域 |
| ArcRepr | 共享（Arc） | 是 | 通过 make_mut() 写时复制 | 浅拷贝（引用计数+1） | 跨线程共享、延迟复制 |

**Owned 分配要求**：

- 须使用 64 字节（或更大）对齐的堆分配
- 须正确处理零大小类型（ZST）和空数组的边界情况（不触发未定义行为）
- 容量为实际分配的元素槽位数（含 padding），与逻辑元素数不同

**各存储模式的 trait 实现关系**：

| 存储模式 | `Storage` | `StorageMut` | `manages_memory()` | `Clone` |
|----------|-----------|--------------|---------------------|---------|
| Owned | ✅ | ✅ | true（独占） | 深拷贝 |
| ViewRepr | ✅ | ❌ | false | O(1) 拷贝元数据 |
| ViewMutRepr | ✅ | ✅ | false | 不可 Clone |
| ArcRepr | ✅ | ❌ | true（共享） | 浅拷贝（Arc ref +1） |

### 6.3 ArcRepr 语义

**ArcRepr::make_mut() 需求**：

| 属性 | 要求 |
|------|------|
| 写时复制 | 若引用计数 > 1，须深拷贝数据到新分配 |
| 线程安全 | 引用计数操作须原子化，多线程并发 make_mut() 不得导致数据竞争 |
| 返回值 | 须返回独占可变访问 |
| 分配保持 | 新分配须保持与原缓冲区相同的对齐值（至少 64 字节），容量须 ≥ 原缓冲区容量，shape/strides/flags 不变 |
| 独占快速路径 | 引用计数为 1 时须直接返回可变引用，无需拷贝 |

**ArcRepr 与视图的安全边界**：

ArcTensor 创建的视图生命周期须绑定到 `&ArcTensor` / `&mut ArcTensor` 自身（而非底层 Arc 数据），确保 make_mut() 与活跃视图之间的互斥由编译器静态保证。

**CowRepr<'a, A>** — Copy-on-Write 存储表示。当前版本仅在 `ensure_blas_compatible()` 返回值中使用（见 §25.4.5），不实现 Storage/StorageMut trait。v2+ 计划实现完整 Storage/StorageMut trait。

### 6.4 设备扩展性

Storage / StorageMut trait 预留 `type Device` 关联类型，当前版本仅支持 `Cpu`。

**`Device` trait**：标记 trait，标识存储所在的计算设备，须满足 `Debug + Clone + Eq + Send + Sync`。当前仅提供 `Cpu` 实现。

**当前版本约束**：

- 设备信息通过 `S::Device` 关联类型承载，`TensorBase<S, D>` 不增加独立泛型参数
- 形状、步长、布局标志为纯元数据，与存储位置无关，可跨设备复用

### 6.5 禁止事项

- 不可将任何 GPU 相关类型或 trait 引入公开 API
- 不可在 `Storage` 或 `StorageMut` trait 中添加 `Gpu` 相关的关联类型或方法（除 `type Device` 占位外）
- 不可在 `Element` trait 中添加 GPU 语义的约束

## 7. 内存布局

### 7.1 布局规则

| 规则 | 要求 |
|------|------|
| 布局顺序 | 仅支持 F-order（列优先），F-order 为项目全局默认 |
| 步长类型 | 有符号类型，单位为元素个数（非字节），支持负步长 |
| 默认对齐 | 自有存储默认 64 字节对齐（AVX-512 缓存行） |
| 可配置对齐 | 构造时可指定对齐值（须为 2 的幂，≥ 元素自然对齐） |
| 填充轴 | F-order 填充 axis 0（第一轴），确保每列起始地址对齐 |
| 填充初始值 | 填充区域零初始化 |

### 7.2 Order

内存布局顺序，用于构造函数和操作中指定或查询布局方向。当前仅 F-order。各 API 中 order 参数传 None 时使用默认 F-order。类型设计须允许未来扩展新布局顺序。

### 7.3 布局状态追踪

须缓存以下布局状态，避免运行时重复计算：

| 状态 | 说明 |
|------|------|
| F-order 连续性 | 是否 F-order 严格连续 |
| SIMD 对齐 | 数据起始地址是否满足对齐要求 |
| 零步长 | 是否存在步长为 0 的维度（广播或 unsqueeze 产生）。空数组时为 false |
| 负步长 | 是否存在负步长（反转维度） |
| 主维度填充 | 主维度是否经过填充 |
| 数据所有权 | 是否独占拥有底层数据（仅 Owned 为 true，ArcRepr 为 false） |
| F-order 宽松连续 | F-order 宽松连续（允许主维度填充） |

布局状态须在以下时机更新：创建时初始化、切片/转置/reshape 后重新计算、视图创建时继承并按需降级。

### 7.4 布局查询

TensorBase 须提供以下布局查询能力：

| 查询能力 | 说明 |
|----------|------|
| 严格 F-contiguous | 步长严格等于 shape 乘积（不含填充） |
| 宽松 F-contiguous | 允许主维度填充的 F-order 连续性 |
| 逐轴连续 | 指定轴是否连续（步长绝对值为 1） |
| SIMD 对齐 | 数据是否满足对齐要求 |
| 零步长检测 | 是否存在零步长维度 |
| 负步长检测 | 是否存在负步长 |
| 填充检测 | 是否存在主维度填充 |

### 7.5 对齐策略

| 属性 | 要求 |
|------|------|
| 默认对齐 | 64 字节（AVX-512 缓存行） |
| 小数组优化 | 当元素总字节 ≤ 64 时，允许降级到元素自然对齐（至少 8 字节），降级后对齐标志为 false |
| 对齐查询 | 须提供方法返回当前数组的实际对齐值 |
| 视图对齐 | 视图继承源数组对齐状态；切片后对齐可能降级，须反映实际对齐状态 |
| ArcRepr 对齐 | make_mut() 触发复制时，新分配使用默认对齐 |

### 7.6 填充语义

#### 7.6.1 填充轴与步长

F-order 填充 axis 0（第一轴）。填充后步长须反映实际物理内存布局（含填充量）。1D 数组无需填充。

#### 7.6.2 填充量计算

填充量须将主维度逻辑元素数向上取整到对齐字节边界。计算过程须使用 checked 算术，溢出时返回 `ShapeError::Overflow`。最终物理字节数须 ≤ `isize::MAX`。对齐值须为元素大小的整数倍。

#### 7.6.3 填充相关构造

- 标准构造函数默认不填充（无额外内存开销）
- 须提供创建填充布局零数组的能力
- 须提供将已有 Tensor 转换为填充布局的能力（需拷贝）

#### 7.6.4 操作后填充行为

| 操作 | 填充保持 | 说明 |
|------|----------|------|
| 逐元素运算 | 不保持 | 输出使用标准无填充分配 |
| reshape | 不保持 | 连续路径保持连续但移除填充 |
| 切片 | 可能保持 | 取决于是否保留填充轴 |
| 转置 | 保持 | 填充轴改变 |
| clone / to_owned | 保持 | 深拷贝保留填充布局 |
| 归约 | 不保持 | 输出为低维数组 |
| make_mut (ArcRepr) | 保持 | 深拷贝时保持 strides 和填充标志 |

#### 7.6.5 填充字节的值约束

| 规则 | 说明 |
|------|------|
| 初始化 | 填充区域零初始化 |
| 逐元素写操作 | 只写入逻辑元素范围，填充字节值变为未定义 |
| SIMD 读取 | 读取填充字节安全（无副作用） |
| SIMD 写入 | 写操作可能覆写填充字节，覆写后值未定义 |
| 逻辑元素访问 | 填充字节不得作为逻辑元素暴露给用户 |
| SIMD tail 处理 | SIMD 操作须按逻辑元素边界处理尾部 |

#### 7.6.6 填充与连续性关系

须支持区分以下连续性概念：

| 概念 | 适用场景 |
|------|----------|
| 严格 F-contiguous | reshape 零拷贝判定 |
| 宽松 F-contiguous | SIMD 路径选择（允许主维度步长 ≥ shape 乘积） |
| 逐轴连续 | 沿指定轴 SIMD 处理 |

Padded 数组非严格连续，reshape 默认触发数据拷贝。需要在 padded 数组上执行零拷贝 reshape 时，须先去除填充（需拷贝数据），再执行 reshape。

## 8. 张量核心抽象

### 8.1 泛型设计

核心数据结构为双参数泛型 `TensorBase<S, D>`：
- S：存储模式（见 §6）
- D：维度类型（见 §3）

**TensorBase 内部组成**：

| 组件 | 说明 |
|------|------|
| storage: S | 底层数据存储（决定所有权与访问权限） |
| shape: D | 各轴长度 |
| strides: D | 各轴步长（元素单位，有符号语义，见 §3.2） |
| offset: usize | 数据起始偏移量（元素单位，非字节） |
| flags: u16 | 布局标志位（见 §7.3） |

**指针语义**：须区分缓冲区起始指针（`storage.as_ptr()`）与数据起始指针（含 offset 偏移）。元素地址通过数据起始指针加各轴索引与步长的加权和计算。

**安全性要求**：所有构造路径须验证 shape、strides 和 offset 的合法性，确保：

- shape 各轴不超过 `isize::MAX`
- 步长累积乘积不溢出
- 元素地址计算始终落在物理缓冲区范围内（含负步长的下界检查）
- `shape.size_checked()` 返回 `Some`（总元素数不溢出 usize）

验证逻辑须统一，确保所有构造路径（包括 `from_raw_parts`、视图构造等）的验证规则一致。验证过程中涉及步长与 shape 乘积的计算须使用检查算术，防止溢出导致的越界访问。

### 8.2 类型别名体系

**主类型别名**

| 别名 | 展开 | 说明 |
|------|------|------|
| Tensor<A, D> | TensorBase<Owned<A>, D> | 拥有数据的数组 |
| TensorView<'a, A, D> | TensorBase<ViewRepr<&'a A>, D> | 不可变视图 |
| TensorViewMut<'a, A, D> | TensorBase<ViewMutRepr<&'a mut A>, D> | 可变视图 |
| ArcTensor<A, D> | TensorBase<ArcRepr<A>, D> | 原子引用计数共享（内置写时复制） |

**维度便捷别名**（以 Tensor 为例，其他存储模式同理）

| 别名 | 展开 | 说明 |
|------|------|------|
| Tensor0<A> | Tensor<A, Ix0> | 0 维标量 |
| Tensor1<A> | Tensor<A, Ix1> | 1 维向量 |
| Tensor2<A> | Tensor<A, Ix2> | 2 维矩阵 |
| Tensor3<A> | Tensor<A, Ix3> | 3 维 |
| Tensor4<A> | Tensor<A, Ix4> | 4 维 |
| Tensor5<A> | Tensor<A, Ix5> | 5 维 |
| Tensor6<A> | Tensor<A, Ix6> | 6 维 |
| TensorD<A> | Tensor<A, IxDyn> | 动态维度 |

### 8.3 基础查询方法

所有 TensorBase<S, D> 均提供以下查询方法（不区分存储模式）：

**维度与形状查询**

| 方法 | 签名 | 说明 |
|------|------|------|
| `len()` | `&self -> usize` | 总元素数（各轴长度乘积），溢出时 wrapping |
| `len_checked()` | `&self -> Option<usize>` | 总元素数，溢出时返回 None |
| `is_empty()` | `&self -> bool` | 是否为空数组 |
| `ndim()` | `&self -> usize` | 维度数 |
| `shape()` | `&self -> &[usize]` | 各轴长度 |
| `strides()` | `&self -> &[isize]` | 各轴步长（有符号，元素单位） |
| `stride_at()` | `&self, axis: usize -> isize` | 指定轴步长（有符号） |
| `offset()` | `&self -> usize` | 数据起始偏移量（元素单位） |

**指针查询**（亦用于 FFI，详见 §25.1）

| 方法 | 签名 | 说明 |
|------|------|------|
| `as_ptr()` | `&self -> *const A` | 数据起始位置不可变指针 |
| `as_mut_ptr()` | `&mut self -> *mut A` | 数据起始位置可变指针（须可写存储） |
| `as_ptr_unchecked()` | `unsafe &self -> *const A` | 不检查偏移量的指针变体 |
| `capacity()` | `&self -> usize` | 物理缓冲区容量（含 padding，元素单位） |

**布局查询**（基于 flags 字段的 O(1) 查询，标志定义见 §7.3）

| 方法 | 签名 | 说明 |
|------|------|------|
| `is_f_contiguous()` | `&self -> bool` | 是否 F-order 严格连续 |
| `is_contiguous()` | `&self -> bool` | 等价 `is_f_contiguous()` |
| `is_f_padded_contiguous()` | `&self -> bool` | 是否 F-order 宽松连续（允许填充） |
| `is_padded_contiguous()` | `&self -> bool` | 等价 `is_f_padded_contiguous()` |
| `is_aligned()` | `&self -> bool` | 是否 SIMD 对齐 |
| `has_zero_stride()` | `&self -> bool` | 是否存在零步长 |
| `has_neg_stride()` | `&self -> bool` | 是否存在负步长 |
| `is_padded()` | `&self -> bool` | 是否存在主维度填充 |
| `layout_flags()` | `&self -> u16` | 返回完整布局标志 |

各布局查询方法的详细判定规则见 §7.4。

# 第三部分：计算模型

## 9. 计算后端

### 9.1 SIMD

#### 9.1.1 基本配置

| 属性 | 要求 |
|------|------|
| 启用方式 | 通过 feature gate `simd` 启用，默认关闭 |
| 实现方式 | 使用 pulp crate 抽象 SIMD 指令集，不直接使用 std::arch |

#### 9.1.2 支持的指令集

运行时自动检测最优指令集并分发到对应实现。不支持任何 SIMD 指令集时退化为标量路径。

| 目标架构 | 支持的指令集 | 说明 |
|----------|-------------|------|
| x86_64 | AVX-512（须显式启用 `simd-avx512` 子 feature）、AVX2、标量 | 不支持 AVX2 的 CPU 退化为标量 |
| aarch64 | NEON、标量 | — |


#### 9.1.3 SIMD 适用操作

**SIMD 路径的通用前提条件**（适用于以下所有 SIMD-eligible 操作）：

| 条件 | 说明 |
|------|------|
| 宽松连续内存 | `is_padded_contiguous()` 为 true（见 §7.4） |
| 正步长方向 | 所有步长须为正值（size-1 维度步长可为 0），负步长数组须回退标量路径 |
| 对齐要求 | 见 §9.1.4 |

**适用 SIMD 的操作**：

| 操作类别 | 示例 | 额外 SIMD 要求 |
|----------|------|---------------|
| 逐元素一元 | abs, neg, sqrt, exp, ln | 无额外要求 |
| 逐元素二元（浮点/复数/整数） | add, sub, mul, div | 两操作数均满足通用前提，形状兼容，且布局方向一致（同为 F-padded）。任一条件不满足时回退标量路径 |
| 逐元素二元（整数除法） | div（整数类型） | 不走 SIMD 路径，始终标量 |
| 归约（浮点/复数） | sum | 满足通用前提即可 |
| 归约（整数 sum） | sum | i32 使用 widening 累加的 SIMD 路径；i64/u64 因无更大标准整数类型，回退标量 checked 路径（见 §14.3） |
| 内积 | dot | 满足通用前提即可 |
| 比较 | eq, lt, gt | 满足通用前提即可 |

**不适用 SIMD 的操作**（即使连续内存也走标量路径）：

| 操作 | 原因 |
|------|------|
| 整数除法（i32/i64） | 主流 SIMD 指令集不提供整数除法指令 |
| 超越函数（sin, exp, ln 等） | pulp 不提供 SIMD 超越函数 |

**广播操作数的 SIMD 优化**：当二元运算的操作数之一为广播视图时，须提供 SIMD broadcast 优化路径。

#### 9.1.4 SIMD 对齐与回退

| 属性 | 要求 |
|------|------|
| 对齐加载 | 推荐直接使用非对齐加载，不做运行时对齐检查 |
| 非连续数据 | 不满足 `is_padded_contiguous()` 时回退到标量路径 |
| 负步长数据 | 任一轴步长为负值时，须回退标量路径 |
| 二元布局不兼容 | 两操作数布局方向不一致（非同为 F-padded）时回退标量路径 |
| 填充字节处理 | SIMD 读写填充区域字节是安全的，但不影响逻辑元素 |
| 尾部处理 | 元素数非 SIMD 宽度整数倍时，尾部使用标量处理，按逻辑元素边界处理 |

### 9.2 并行

#### 9.2.1 基本配置

| 属性 | 要求 |
|------|------|
| 启用方式 | 通过 feature gate `parallel` 启用，默认关闭 |
| 实现方式 | 使用 rayon crate 实现数据并行 |

#### 9.2.2 并行策略

| 属性 | 要求 |
|------|------|
| 并行阈值 | 默认数据量（`元素数 × size_of::<A>()`）≥ 256KB 时启用并行，可通过全局配置或单次调用覆盖。配置须跨线程生效（全局阈值须对 rayon 工作线程可见） |
| 支持并行的操作 | 逐元素运算、归约、map/mapv、zip 迭代 |
| 不支持并行的操作 | 矩阵乘法、单元素操作、小数组操作 |
| 分块策略 | 按连续内存块分割，每块不小于 4K 元素。分割边界须对齐到 64 字节以避免 false sharing。非连续数组须选择连续轴优先分块 |
| 嵌套并行 | 禁止嵌套并行（内层操作强制单线程），避免线程池饥饿 |
| 线程数 | 默认使用 rayon 全局线程池，可通过自定义线程池覆盖 |
| 浮点归约 | 须使用补偿求和算法以保证精度。单线程路径使用 Neumaier 补偿求和；并行路径各线程独立累加后汇总。整数归约使用 widening 累加（见 §9.1.3），不使用补偿求和 |

### 9.3 性能分层

运行时根据数据规模、内存布局和步长方向自动选择执行路径。须按以下优先级（首个匹配的行即为执行路径）：

| 优先级 | 条件 | 执行路径 |
|--------|------|----------|
| 1 | 元素数 < SIMD 宽度 | 标量路径 |
| 2 | 满足 SIMD 条件 且 元素数 < 并行阈值（或 parallel 未启用） | SIMD 路径（单线程） |
| 3 | 满足 SIMD 条件 且 元素数 ≥ 并行阈值 且 parallel 启用 | SIMD + 并行路径 |
| 4 | 不满足 SIMD 条件 且 元素数 ≥ 并行阈值 且 parallel 启用 | 标量 + 并行路径 |
| 5 | 不满足 SIMD 条件 且 元素数 < 并行阈值（或 parallel 未启用） | 标量路径 |

其中"满足 SIMD 条件"指：`is_padded_contiguous()` 为 true、所有步长为正（size-1 维度步长可为 0）、`simd` feature 已启用、当前操作存在 SIMD 实现。

**二元操作的 dispatch 扩展**：二元操作（如 `a + b`）须在进入上述分层表之前执行额外的前置检查：

| 检查阶段 | 条件 | 执行路径 |
|----------|------|----------|
| 布局兼容性 | 两操作数布局方向不一致 | 标量路径 |
| 相同形状 | `a.shape() == b.shape()` | 跳过广播计算，直接进入存储复用 |
| 广播 SIMD | 一方连续+正步长 且另一方 zero-stride | 广播 SIMD 路径 |
| 存储复用 | 输出可复用某操作数存储（所有权/对齐/容量满足） | 就地写入 |
| 分层回退 | 按输出数组属性匹配上述 5 级分层表 | 对应层级路径 |

存储复用优先级：优先复用左侧操作数（`a`）的存储，次选右侧操作数（`b`），均不满足时新分配存储。

## 10. 线程安全

### 10.1 Send/Sync 保证

所有存储模式须根据 Rust 的 `Send`/`Sync` 规则自动推导线程安全性。各存储模式须满足：Owned 和 ArcRepr 为 `Send + Sync`（元素类型为 `Send + Sync` 时）；ViewRepr 为 `Send + Sync`（元素类型为 `Send + Sync` 时）；ViewMutRepr 为 `Send` 但非 `Sync`（元素类型为 `Send` 时）。

### 10.2 并行迭代安全

并行迭代须保证各线程访问不重叠的元素区间，非连续数组须正确处理步长跳跃，可变并行迭代须保证独占访问（无别名）。

### 10.3 Padding 字节并发规则

Padding 字节不得作为逻辑元素暴露给用户。并行迭代须按逻辑元素边界分割工作区间，确保 SIMD 写入 padding 字节时不产生数据竞争。逐元素写操作只写入逻辑元素范围。

## 11. 迭代器

### 11.1 迭代类型

须支持以下迭代类型：

| 类型 | 说明 |
|------|------|
| 元素迭代 | 遍历所有元素 |
| 轴迭代 | 沿指定轴，每次产出降维子视图 |
| 窗口迭代 | 滑动窗口 |
| 索引迭代 | 带多维索引的元素迭代 |
| 并行迭代 | 数据并行迭代（feature `parallel`） |
| 多数组同步迭代 | zip 多数组同步遍历 |
| 有序迭代 | 可指定遍历顺序（F/C） |

### 11.2 迭代器协议

| 场景 | 行为 |
|------|------|
| zip 形状完全一致 | 按指定遍历顺序同步产出元素 |
| zip 形状不一致但可广播 | 自动广播后迭代，无额外数据拷贝 |
| zip 形状不一致且不可广播 | 返回错误 |
| 遍历顺序未指定 | 使用默认遍历顺序 |
| 遍历顺序指定为 F/C | 强制按指定逻辑顺序 |
| 窗口迭代越界 | 不产出不完整窗口 |
| 空数组迭代 | 立即结束，产出零个元素 |

广播迭代不产生实际数据拷贝。

### 11.3 广播视图的可变迭代安全性

禁止对广播视图进行可变迭代。广播视图存在多个逻辑位置映射到同一物理元素的情况，可变迭代会导致别名写入，违反 Rust 别名规则。此规则不适用于只读迭代。

### 11.4 迭代器通用要求

| 要求 | 说明 |
|------|------|
| 精确长度 | 所有迭代器须提供精确的元素数量信息 |
| 已知长度 | 元素迭代、轴迭代、索引迭代的长度在迭代前已知 |
| 双向遍历 | 连续数组的元素迭代器支持双向遍历 |
| 熔断 | 迭代结束后不再产出元素 |
| 按值与借用 | 所有张量类型支持按值消费和借用两种迭代方式 |
| 并行支持 | 所有张量类型支持并行迭代（feature `parallel`），元素类型须可跨线程安全传递 |

### 11.5 元素迭代

须提供不可变和可变两种变体。须支持非连续数组的通用遍历和连续数组的优化遍历。填充数组的迭代须仅遍历逻辑元素，不暴露填充区域。可变迭代要求可写存储，广播视图的可变迭代须被禁止（见 §11.3）。

### 11.6 轴迭代器

须提供沿指定轴遍历的迭代器，每次产出降维子视图（维度减少 1）。须同时提供不可变和可变变体，可变变体要求可写存储。每个子视图共享源数组底层存储（零拷贝）。轴索引超出范围时 panic。轴长度为 0 时迭代器立即结束。零维张量不支持轴迭代。

### 11.7 索引迭代器

须提供带多维索引的元素迭代，每次同时产出逻辑坐标和元素引用。索引始终按 F-order 逻辑坐标递增。须提供不可变和可变变体。

### 11.8 并行迭代

须提供并行元素迭代，将元素按连续块分割给线程。须支持所有存储模式，ArcRepr 不支持可变并行迭代。不可变迭代要求元素类型可跨线程共享，可变迭代要求元素类型可跨线程移动。

### 11.9 窗口迭代器

须支持任意步幅的源数组。

**边界行为**：

| 场景 | 行为 |
|------|------|
| 窗口尺寸某轴 > 源数组对应轴 | 产出零个窗口 |
| 窗口尺寸 == 源数组形状（所有轴） | 产出恰好 1 个窗口 |
| 源数组某轴长度为 0 | 产出零个窗口 |
| 窗口尺寸 ndim != 源数组 ndim | panic |

### 11.10 零维张量迭代

零维张量元素数 = 1，元素迭代产出恰好 1 个元素。不支持轴迭代和窗口迭代。

## 15. 集合操作

须提供唯一值操作（unique），返回排序后的不重复元素。

**适用元素类型**：整数、浮点、复数。bool 和 usize 不适用。

**多维输入处理**：任意维度的输入数组按 F-order 展平为一维后操作。连续输入须零拷贝展平；非连续输入需拷贝。

**排序与去重要求**：须为所有适用类型定义全序比较规则——整数使用自然全序，浮点使用 IEEE 754 全序语义，复数使用字典序（先实部后虚部）。NaN 须视为相等并合并为一个，排在末尾。+0.0 与 -0.0 须视为相同值。

空数组返回空的一维数组。须同时提供按自定义键去重的能力，供用户自定义等价判定规则。

## 16. 广播

### 16.1 基本规则

遵循 NumPy 广播规则（右对齐，size-1 维度拉伸）。广播为零拷贝操作，不产生实际数据拷贝。

**广播规则**：

| 规则 | 说明 |
|------|------|
| 维度对齐 | 从最右维度开始对齐，维度数不足的数组在左侧逻辑补 1 |
| 兼容检查 | 对应维度相等，或其中一个为 1。不满足时返回广播错误 |
| 输出形状 | 每个轴取两个输入中非 1 的值；均为非 1 时须相等。结果轴大小取非 1 的值（而非取最大值），这对 size-0 维度至关重要 |
| 零拷贝实现 | 广播通过步长机制模拟维度扩展，不复制数据 |

**设计约束**：

| 约束 | 说明 |
|------|------|
| 广播视图为只读 | 广播产生的视图始终为只读（写入语义不明确） |
| 算术运算符隐式支持广播 | 二元运算符自动触发广播（见 §12.6、§20） |
| 标量广播 | 标量视为 0 维数组，可与任意维度数组广播 |
| 原地广播运算 | 左操作数不可被广播，须为连续内存，确保每个逻辑位置有唯一的物理写入目标 |

### 16.2 显式广播

须提供显式广播方法，接受目标形状参数，返回零拷贝广播视图。目标形状与源形状须按 §16.1 规则兼容，不兼容时返回广播错误（包含冲突轴信息）。视图的生命周期绑定到源数组引用。

### 16.3 边界行为

- **0D（标量）广播**：0D 数组可广播到任意目标形状，所有轴均为广播维度
- **含 size-0 维度的广播**：size-0 与 size-1 兼容（结果为 0），size-0 与 size-0 兼容（结果为 0），size-0 与 size-N（N>1 且 N≠0）不兼容。广播结果为空数组时，无实际计算开销
- **广播后维度数 > 6**：须使用动态维度类型
- **维度类型一致**：运算符要求两操作数维度类型相同，不产生维度提升。不同维度类型须先显式统一

### 16.4 可恢复广播运算

二元运算符在广播形状不兼容时只能 panic。须同时提供可恢复的错误处理路径，在广播失败时返回错误而非 panic。该路径的运算语义与对应运算符完全一致，仅错误处理方式不同。

## 17. 形状操作

### 17.1 转置

须提供全轴反转转置和指定轴排列转置两种操作。转置为始终零拷贝操作，仅重排步长和形状元数据，不改变维度数。

全轴反转转置：反转所有轴顺序，一维和零维数组为无操作。

指定轴排列转置：按给定排列重排轴。排列须为目标维度范围内的合法排列（无重复、无遗漏），否则 panic。

## 18. 索引操作

须支持以下索引类型：多维整数索引（逐元素定位）、范围索引（切片）、布尔掩码高级索引。

## 20. 运算符重载

### 20.1 重载类型

须为四则运算符（加、减、乘、除）提供运算符重载，隐式支持广播。

当前版本不支持比较运算符、复合赋值运算符、一元运算符、位运算符的重载。

### 20.2 二元运算符所有权语义

二元运算符须支持张量与张量、张量与标量之间的运算。存储复用由操作数的所有权语义决定：按值传入（消耗所有权）表示允许复用其存储，按引用传入表示不复用。所有组合均返回 Owned 张量。

存储复用的前提为被复用的操作数具有连续内存且形状与输出一致（含广播展开后），不满足时须静默回退到分配新数组。

二元运算符要求两操作数的维度类型相同。不同维度类型的运算须先统一为动态维度。

张量-标量运算须支持标量在左操作数和右操作数两种位置。

## 22. 连续性保证

须提供连续性保证操作，确保张量数据在内存中按 F-order 连续排列。须提供以下三类操作：

- 始终返回新的连续布局张量（引用输入，数据拷贝）
- 零拷贝获取连续视图（仅当已满足连续性时返回视图，否则返回空）
- 消耗所有权并按需拷贝（已连续时零拷贝返回原张量，否则拷贝为连续布局的新张量）

连续性判定使用严格连续性。填充数组和广播视图不满足严格连续性。

须同时提供宽松连续性变体，允许填充张量在 SIMD 运算、BLAS FFI 等能正确处理填充的场景下零拷贝获取视图。

所有连续性保证操作输出的张量须满足：步长为严格连续步长（无填充），连续性标志为 true。

引用输入和零拷贝视图路径须支持所有存储模式；消耗所有权的路径仅适用于 Owned 存储。

各类边界情况（0D 标量、空张量、1D 数组、填充数组、广播视图、非连续视图/Owned）须有明确的处理规则。

## 23. 转换操作

### 23.1 cast 语义

须提供逐元素类型转换方法，消耗 Owned 数组并返回目标元素类型的新数组。目标类型须在本库的类型体系内（§4）。转换始终成功，溢出使用饱和语义。

须同时提供同类型拷贝方法，用于泛型管线中与类型转换对称的调用路径。

仅 Owned 存储模式支持 cast；View 和 ArcRepr 须先转为 Owned 再执行转换。

不支持跨类别隐式转换（如复数→实数须显式取实部）。

**精度行为**：

须为所有支持的源→目标类型组合定义明确的转换语义，包括：浮点间精度升降、浮点→整数（截断+溢出饱和）、整数→浮点（舍入）、整数间宽度变化（窄化饱和、宽化精确）、特殊浮点值（NaN→整数为 0、Inf→整数饱和）、bool↔数值、实数↔复数、复数间精度变化。所有精度行为须与 Rust `as` 转换语义一致。

### 23.2 标准类型转换

须支持从常见 Rust 集合类型（Vec、切片、数组）和标量值构造张量。嵌套集合（如 `Vec<Vec<A>>`）的构造须验证内层一致性，不一致时返回错误而非 panic。

须支持静态维度与动态维度之间的互转（见 §3.5）。

须支持各存储模式之间的转换：任意→Owned、Owned→View、Owned→ViewMut、Owned→ArcTensor、ArcTensor→Owned。

须为张量类型实现标准库转换 trait（`From`/`TryFrom`/`AsRef`/`AsMut`/`Borrow`/`Default`），遵循各 trait 的语义约定。

## 24. 格式化输出

### 24.1 设计原则

| 原则 | 说明 |
|------|------|
| Display 面向用户 | 简洁、可读，只显示数据内容，不显示内部实现细节（存储模式、步长、偏移量） |
| NumPy 风格 | 数据部分使用方括号 `[]` 包裹，逗号+空格分隔元素 |

### 24.2 元素级格式化规则

各元素类型在数组格式化中的显示须遵循统一规则。特殊浮点值（NaN、±Inf）须有明确的显示形式。

### 24.3 Display 输出规范

Display 只显示数据内容，不显示 shape、dtype、存储模式、步长或偏移量。

须为不同维度（0D 标量、1D、2D、3D 及以上、空数组）定义明确的输出格式，包含适当的嵌套与缩进。

### 24.4 大数组截断规则

当元素总数超过阈值时须触发截断，避免输出过长。截断时保留首尾部分数据，中间以省略标记替代。当前版本使用固定默认值，不提供用户可配置的截断参数。

### 24.5 空数组与 0D 标量

**空数组**：shape 中任一轴为 0 的数组，数据部分固定为空。

**0D 标量**（shape=[], ndim=0, len()=1）：数据部分直接显示标量值，不附带中括号。

## 25. FFI 集成

### 25.1 指针 API

提供原始指针访问，供上游库零开销集成。须支持不可变指针、可变指针（须可写存储）、unchecked 变体及物理缓冲区容量查询。ArcRepr 的裸指针须注意 `make_mut()` 可能使指针失效。

### 25.2 不安全构造函数

须支持从裸指针构造 Tensor（获取所有权）、只读视图、可变视图，以及将 Tensor 解构为裸指针 + shape + strides + offset + capacity（阻止 Rust 释放内存）。除 `into_raw_parts` 外均为 `unsafe`。

### 25.3 形状与步长查询（字节单位）

须提供字节单位的步长查询和数据起始偏移量查询。

### 25.4 BLAS 兼容性

须提供 BLAS 兼容性查询与布局转换 API：leading dimension 查询、布局兼容性检查、BLAS 布局标识、BLAS 转置参数推断、以及确保 BLAS 兼容的布局转换（已兼容时零拷贝，否则拷贝为连续布局）。须定义 BLAS 兼容条件与相关错误类型。

### 25.5 Trans 枚举

须提供转置标识枚举，包含不转置、转置、共轭转置三种变体。

### 25.6 索引转换

须支持多维索引到元素原始指针和偏移量的转换，包含 checked（越界 panic）和 unchecked（越界为 UB）两种变体。

### 25.7 FFI 使用模式

须支持以下 FFI 使用模式：只读传递给 C 库、可写传递给 C 库、从 C 库接收数据构造 Tensor、BLAS 集成。

## 26. 临时工作空间

须提供临时工作空间类型，供上游库进行对齐分配的临时缓冲区操作。工作空间须支持 `no_std` + `alloc` 环境。

### 26.1 工作空间属性

须支持默认 64 字节对齐及调用方指定更大对齐值。借用语义：借出期间不可再次借出，归还后可复用。须支持自动扩容（保持对齐）及将未使用空间归还给分配器。工作空间不绑定线程，不保证零初始化。

### 26.2 分割与嵌套

须支持从同一工作空间分割多个不重叠的子缓冲区、将工作空间分割为两个独立子工作空间各自借出、以及递归分割（设有最小分割粒度下限）。

### 26.3 线程安全

工作空间须实现 `Send`（可在线程间移动），不实现 `Sync`。通过分割后的子工作空间可分别移动到不同线程独立使用。

### 26.4 扩容安全性

活跃借用期间不允许扩容（由借用生命周期保证）。子工作空间扩容时须创建独立分配，不影响其他子工作空间。

### 26.5 借出与归还

须提供多种借出方式：按字节数借出、按元素类型借出（类型化、按元素类型和工作空间对齐的较大值对齐）、按需初始化（未初始化、指定值初始化、零初始化、源数据拷贝写入）及按 `ScratchRequirement`（描述所需缓冲区大小和对齐要求）借出。借出的引用生命周期结束后自动归还。

### 26.6 查询与管理

须提供当前已分配字节容量查询、剩余可用字节数查询、重置分配指针使已分配空间可重用、以及将未使用空间归还给分配器的能力。

# 第五部分：工程保障

## 27. 错误处理

### 27.1 错误分类

库须定义覆盖以下场景的错误类型体系：

**可恢复错误**（返回 Result）：

| 错误场景 | 说明 |
|----------|------|
| 形状不兼容 | 二元运算/zip 形状不兼容且无法广播 |
| 广播规则不满足 | 非兼容维度无法广播 |
| 布局不满足要求 | 要求连续布局但输入非连续 |
| 轴索引无效 | 轴索引超出维度数 |
| 构造参数形状无效 | 如 cat 输入 ndim 不一致 |
| 维度互转失败 | 静态/动态维度互转时维度数不匹配 |
| 空数组归约 | 对空数组执行要求至少一个元素的归约操作 |
| 参数语义不合法 | 参数值不合法（非结构/索引类问题） |
| 布局/对齐验证失败 | shape × strides 不一致、指针未对齐、对齐值无效等 |
| 嵌套容器长度不一致 | 如 Vec<Vec<A>> 内层长度不一致 |

**不可恢复错误**（panic）：

| 错误场景 | 说明 |
|----------|------|
| 内存分配失败 | 超大数组或系统内存不足 |
| 索引越界 | 编程错误，同时提供 unchecked 变体 |

错误类型之间须有清晰的边界：参数语义不合法、结构不合法、索引超范围等应归为不同错误类型；当问题可归为更具体的类型时，优先使用更具体的类型。

### 27.2 错误诊断信息

所有返回 Result 的错误类型须携带充分的诊断信息，使调用方能定位问题根因。panic 类错误通过 panic message 传递诊断信息。

### 27.3 处理策略

**判断原则**：panic 用于"调用方违反 API 前置条件"（编程错误），Result 用于"输入数据属性导致操作无法完成"（合法调用路径，调用方应处理）。

| 策略 | 适用场景 |
|------|----------|
| 返回 Result | 形状/布局/轴错误、参数语义不合法、维度互转失败等可恢复错误 |
| panic | 索引越界（同时提供 unchecked 变体）、前置条件违反、内存分配失败 |

运算符重载触发的隐式广播失败时 panic；须同时提供返回 Result 的可恢复替代路径。

### 27.4 no_std 兼容

错误类型须兼容 `no_std` 环境，不引入额外依赖。无论哪种环境，内存分配失败时诊断信息均须包含请求的分配大小。

### 27.5 Panic Safety

panic 发生后须保证：(1) 可安全 drop（无 UB、无内存泄漏）；(2) 不保证数据一致性。

并行操作 panic 时：单个线程 panic 不影响其他线程内存安全；panic 须立即传播，不得吞没。所有 drop 实现不得 panic。

## 28. 质量要求

### 28.1 文档

| 要求 | 范围 |
|------|------|
| doc comment | 所有公开 API |
| Safety 文档节 | 所有 unsafe 函数 |
| 使用示例 | 关键 API |

### 28.2 测试

| 要求 | 指标 |
|------|------|
| 测试类型 | 单元测试 + 集成测试 + 边界测试 |
| 行覆盖率（整体） | ≥ 90% |
| unsafe 代码块覆盖率 | ≥ 95%（单独统计） |
| 分支覆盖率（关键路径） | ≥ 85% |

### 28.3 数值精度

**浮点与复数精度要求**：各类运算（加减乘、除法、归约、超越函数）须满足与精度等级对应的误差上界（ULP 或相对误差）。归约精度指标适用于 N ≤ 10⁶；大数组归约须保证数值稳定性。并行浮点归约允许精度容差内差异。

**整数运算正确性**：整数归约溢出时须 panic（debug 和 release 模式一致），不得静默 wrapping。

**精度验证要求**：

| 属性 | 要求 |
|------|------|
| 边界值覆盖 | 0、±1、极大值、极小正规数、subnormal、±Inf、NaN；复数额外覆盖纯虚数、零虚部、零实部、极大模 |
| 回归测试 | 固定种子的 golden test |
| 跨平台一致性 | 同一输入在不同平台结果须在精度容差内一致 |

### 28.4 边界覆盖

须覆盖以下边界情况：
- 空张量、单元素、大张量
- 极端值（NaN/Inf/subnormal）
- 非连续布局
- 高维（≥4维）
- 广播形状不兼容（返回错误）
- ArcRepr 写时复制（引用计数不同时的行为差异）
- 小数组对齐降级
- 填充布局操作后的步长正确性
- IxDyn 与 IxN 互转（匹配/不匹配）
- no_std 构建验证

### 28.5 并行与 SIMD 测试

| 测试类别 | 要求 |
|----------|------|
| 数据竞争检测 | CI 中须包含数据竞争检测 |
| 未定义行为检测 | CI 中须包含未定义行为检测 |
| SIMD 正确性 | SIMD 路径结果须与标量路径在精度容差内一致 |
| 并行确定性 | 并行归约结果须与单线程一致（浮点允许精度容差内差异） |
| 跨平台 SIMD | 不同 SIMD 指令集上结果须一致 |
| 并行边界 | 元素数在并行阈值附近的边界情况 |
